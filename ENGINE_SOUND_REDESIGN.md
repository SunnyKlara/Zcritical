# 引擎音效系统重设计方案

> **目标：** 对标 Forza Horizon 级别的引擎声体验。旋钮调速时引擎声从低沉怠速到高亢嘶吼平滑过渡，油门模式下加速/减速有真实的转速感。

---

## 一、标杆分析

### Forza Horizon 的做法
录制真实引擎在不同 RPM 下的声音（怠速、低转、中转、高转），运行时根据当前转速在相邻层之间交叉淡入淡出 + 变速播放。多达几十层采样 + 排气/进气/涡轮分层混合。

### ESP32 上已验证的方案：Rc_Engine_Sound_ESP32
GitHub 上 288 星的成熟开源项目（TheDIYGuy999），专门在 ESP32 上做引擎声模拟，已被大量 RC 模型使用。核心技术：

- **8-bit 22050Hz 单声道 WAV 采样**（存为 C 头文件数组）
- **可变采样率播放**：通过改变读取步进速率实现音调随 RPM 变化
- **多轨实时混合**：怠速声 + 加速声 + 涡轮声 + 柴油爆震声，4 轨同时混合
- **负载感知音量**：油门位置影响各轨音量比例
- **RPM 惯性模拟**：转速不会瞬间跳变，有加速/减速惯性

**关键点：这个方案在普通 ESP32（单核 240MHz，无 PSRAM）上就能流畅运行。我们的 ESP32-S3 双核 + 2MB PSRAM 性能远超需求。**

## 二、我们的方案

### 架构：3 层音频混合 + 可变采样率

```
┌─────────────────────────────────────────────────────────┐
│                    Engine Sound Mixer                     │
│                                                          │
│  Layer 1: IDLE (怠速声)                                   │
│  ┌──────────────────┐                                    │
│  │ idle.h 采样数组   │──→ 可变速率读取 ──→ ×idle_vol ──┐  │
│  │ 22050Hz 8-bit    │    (RPM 控制步进)                │  │
│  └──────────────────┘                                  │  │
│                                                        │  │
│  Layer 2: REV (加速声)                                  ↓  │
│  ┌──────────────────┐                              ┌──────┐│
│  │ rev.h 采样数组    │──→ 可变速率读取 ──→ ×rev_vol──→│混合器││
│  │ 22050Hz 8-bit    │    (RPM 控制步进)              │ SUM ││
│  └──────────────────┘                              └──┬───┘│
│                                                       │    │
│  Layer 3: TURBO (涡轮声，可选)                         │    │
│  ┌──────────────────┐                                 │    │
│  │ turbo.h 采样数组  │──→ 可变速率读取 ──→ ×turbo_vol─┘    │
│  └──────────────────┘                                      │
│                                                            │
│  RPM ← speed_pct (0-100) + 惯性平滑                        │
│  idle_vol = f(RPM, throttle)  高怠速低加速时响              │
│  rev_vol  = f(RPM, throttle)  高转速高油门时响              │
│  turbo_vol = f(RPM)           高转速时响                    │
│                                                            │
│  混合结果 → 16-bit 上采样 → I2S DMA → MAX98357 → 扬声器    │
└─────────────────────────────────────────────────────────────┘
```

### 工作原理

**可变采样率 = 音调变化：**
- 怠速 RPM：以 1.0x 速率读取采样 → 原始音调（低沉）
- 中速 RPM：以 2.0x 速率读取 → 音调升高一倍（中频嘶吼）
- 高速 RPM：以 3.0x 速率读取 → 音调升高两倍（高亢尖锐）

**怠速/加速交叉淡入淡出 = 音色变化：**
- 低 RPM：idle 音量 100%，rev 音量 0% → 纯怠速声
- 中 RPM：idle 音量 50%，rev 音量 50% → 混合过渡
- 高 RPM：idle 音量 0%，rev 音量 100% → 纯加速声

**RPM 惯性 = 真实感：**
- 旋钮调速后，RPM 不会瞬间跳到目标值
- 加速惯性：RPM 以可调速率爬升（模拟发动机响应延迟）
- 减速惯性：松油门后 RPM 缓慢回落（模拟飞轮惯性）

### 与油门模式的融合

油门模式（三击旋钮进入）：
- 长按旋钮 = 踩油门 → RPM 持续爬升 → 引擎声从怠速拉到嘶吼
- 松开 = 松油门 → RPM 缓慢回落 → 引擎声从嘶吼降回怠速
- 风扇转速跟随 RPM → 风声 + 引擎声同步 = 沉浸感

普通模式（旋钮调速）：
- 旋钮顺时针 → 速度增加 → RPM 跟随上升 → 引擎声变高亢
- 旋钮逆时针 → 速度降低 → RPM 跟随下降 → 引擎声变低沉

## 三、素材要求

需要 2-3 段引擎声采样（参考 Rc_Engine_Sound_ESP32 的格式）：

| 文件 | 内容 | 格式 | 时长 |
|------|------|------|------|
| idle | 稳定怠速声（~800RPM） | 22050Hz 8-bit Mono WAV | 0.5-2秒 |
| rev | 稳定高转声（~6000RPM） | 22050Hz 8-bit Mono WAV | 0.5-2秒 |
| turbo（可选） | 涡轮增压呼啸声 | 22050Hz 8-bit Mono WAV | 0.5-1秒 |

**关键要求：**
1. idle 和 rev 的采样长度必须相同（通过 Audacity 调整 Rate 实现）
2. 每段必须是完整的气缸循环（比如 V8 = 8 个点火脉冲为一个循环）
3. 首尾零交叉点对齐，确保无缝循环
4. 最响的峰值放在采样末尾（Rc_Engine_Sound_ESP32 的约定）

**素材来源方案：**
- **方案 A（推荐）：** 直接使用 Rc_Engine_Sound_ESP32 项目自带的几十种引擎声采样。它们已经是处理好的 .h 头文件格式，包括 V8、V12、V6、涡轮柴油等各种引擎。GPL 协议，可以使用。
- **方案 B：** 从 YouTube 引擎声视频中截取稳定 RPM 段，用 Audacity 处理成所需格式
- **方案 C：** 用 enginesound（GitHub DasEtwas）程序化生成引擎声

## 五、技术参数

```c
// 采样格式
#define ENGINE_SAMPLE_RATE    22050   // Hz
#define ENGINE_SAMPLE_BITS    8       // unsigned 8-bit (128 = 零点)

// RPM 范围
#define ENGINE_RPM_IDLE       800     // 怠速转速
#define ENGINE_RPM_MAX        8000    // 最高转速
#define ENGINE_RPM_REV_START  2000    // rev 声开始混入的 RPM

// 惯性（每 20ms 更新一次）
#define ENGINE_ACCEL_RATE     60      // RPM/tick 加速速率
#define ENGINE_DECEL_RATE     40      // RPM/tick 减速速率（松油门回落更慢）

// 音量
#define ENGINE_IDLE_VOL_MAX   180     // idle 最大音量 (0-255)
#define ENGINE_REV_VOL_MAX    255     // rev 最大音量 (0-255)

// I2S 输出
#define ENGINE_OUTPUT_RATE    44100   // 上采样到 44100Hz 输出（匹配 drv_audio）
```

## 六、内存预算

```
idle 采样 (~1秒 22050Hz 8-bit): ~22KB
rev 采样 (~1秒 22050Hz 8-bit):  ~22KB
turbo 采样 (可选):               ~11KB
总 Flash 占用:                   ~55KB（比当前 engine.mp3 的 472KB 还小）

运行时 RAM: 几乎为零（直接从 Flash const 数组读取）
CPU: 单核 <5%（只是数组读取 + 乘法 + 加法）
```

## 八、效果预期

做完后的体验：

1. **进入 Speed 界面** → 引擎启动声（可选的 cranking 音效）→ 怠速低沉"突突突"
2. **旋钮顺时针加速** → 引擎声音调逐渐升高，从低沉变为中频嘶吼，再到高亢尖锐
3. **旋钮逆时针减速** → 引擎声缓慢回落，有飞轮惯性感
4. **油门模式长按** → RPM 持续爬升，引擎声从怠速一路拉到红线区
5. **油门模式松开** → RPM 缓慢回落，引擎声逐渐降回怠速
6. **退出 Speed 界面** → 引擎声淡出消失

这不是"播放一段 MP3 然后改音量"，而是**实时合成的引擎声**，和赛车游戏的原理一样。
