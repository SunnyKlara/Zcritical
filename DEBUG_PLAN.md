# Critical 真机调试规划

> **核心原则：你是我的眼睛，我是你的手。你测试反馈现象，我定位修代码。**
> **最后更新：** 2026-04-29

---

## 工作流程

```
你操作硬件/APP → 发现问题 → 告诉我现象（越具体越好）→ 我读代码定位 → 我改代码 → 你重新烧录/编译验证
```

反馈模板（复制粘贴用）：
```
测试项：H05（旋钮顺时针）
现象：转了没反应 / 反方向 / 跳两格 / 偶尔卡顿
预期：菜单项向下移动
串口日志（如果有）：粘贴关键几行
```

---

## 调试阶段

一共 5 轮，每轮聚焦一个层面。**严格按顺序来**，因为后面的功能依赖前面的基础。

---

### 第 1 轮：硬件独立验证（不连 APP）

**目标：确认 ESP32 硬件本身能正常工作。**

这是地基，如果硬件本身有问题，连 APP 测只会更混乱。

#### 测试顺序：

**Step 1 — 开机基础**
- [ ] H01: 上电，LCD 有没有画面？画面内容对不对？
- [ ] H02: 扬声器有没有声音？是引擎声还是杂音？
- [ ] H04: 手机蓝牙设置里能不能扫到 "T1"？

**Step 2 — 编码器（最核心的输入设备）**
- [ ] H05: 顺时针转，菜单往哪个方向动？
- [ ] H06: 逆时针转，方向对不对？
- [ ] H07: 单击，能进入子菜单吗？
- [ ] H08: 长按，能返回吗？
- [ ] H09: 三击，油门模式能触发吗？

**Step 3 — 菜单 UI 逐页验证**
- [ ] H10-H16: 依次进入 Speed/Preset/RGB/Bright/Volume/Logo/Menu 每个页面
  - 画面显示正常吗？（图标、文字、位置）
  - 旋钮调节有反应吗？
  - 数值变化方向对吗？

**Step 4 — 输出设备**
- [ ] H17: RGB 页面调色，4 路灯带是不是独立变化？哪路不对？
- [ ] H18: 预设切换，LED 颜色对不对？
- [ ] H19: 亮度调节有效果吗？
- [ ] H21-H23: 速度页面调节，风扇转不转？转速跟数值匹配吗？
- [ ] H24-H25: 三击后雾化器开了吗？再三击关了吗？
- [ ] H26-H28: 音量调节，声音大小有变化吗？

**Step 5 — NVS 持久化**
- [ ] H03: 调好预设和亮度后断电重启，恢复了吗？

> **第 1 轮完成标准：** 不连 APP 的情况下，旋钮能正常操作所有菜单，LED/风扇/雾化器/音频都能响应。

---

### 第 2 轮：BLE 连接 + 基础通信

**目标：APP 能连上硬件，基本命令能跑通。**

#### 前置条件
- 第 1 轮硬件独立验证通过（至少编码器和 LED 正常）
- APP debug APK 已安装

#### 测试顺序：

**Step 1 — 连接建立**
- [ ] A05-A08: APP 扫描能找到 T1 吗？
- [ ] B01: 点击连接，能连上吗？连接耗时大概多久？
- [ ] B02-B03: 连接后 APP 界面显示的状态和硬件实际一致吗？
  - 风扇速度、亮度、预设编号、音量、雾化器状态

**Step 2 — 最简单的命令验证**
- [ ] 在 APP Running Mode 调速度滚轮 → 风扇转速有变化吗？
- [ ] 在 APP 切换预设 → LED 颜色变了吗？
- [ ] 在 APP 调亮度 → LED 亮度变了吗？
- [ ] 在 APP 开关雾化器 → 雾化器响应了吗？

**Step 3 — 反向同步（硬件→APP）**
- [ ] R16: 硬件旋钮调速度 → APP 速度滚轮跟着动吗？
- [ ] C05: 硬件旋钮切预设 → APP 胶囊条跟着滚吗？
- [ ] R17: 硬件单击切单位 → APP 单位标签变了吗？

**Step 4 — 断连重连**
- [ ] B10: 拔掉 ESP32 电源 → APP 弹断连对话框了吗？
- [ ] B11: 重新上电后点重连 → 能连回来吗？

> **第 2 轮完成标准：** APP 能连接、发命令、收反馈，断线能重连。

## 注意事项

1. **串口日志很重要** — 测硬件时尽量连着串口监视器（`idf.py monitor`），很多问题从日志就能定位
2. **一次只测一个东西** — 不要同时测多个功能，出了问题不好判断是哪里的
3. **先粗后细** — 第 1 轮只看"能不能用"，不纠结"好不好用"
4. **随时可以暂停** — 测累了就停，把当前进度告诉我，下次继续
