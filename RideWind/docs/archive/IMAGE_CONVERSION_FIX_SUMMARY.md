| 扫描顺序 | 从上到下 | 从下到上 | ❌ 不匹配 |

| 字节序 | 大端序 | 大端序 | ✅ 正确 |
| 位运算 | 正确 | 正确 | ✅ 正确 |

**结论**: 只有扫描顺序错误，其他都是正确的！

## 🔧 技术细节

### BMP格式特点
1. **颜色顺序**: BGR（不是RGB）
2. **扫描顺序**: 从下到上（height > 0时）
3. **行对齐**: 每行字节数必须是4的倍数

### RGB565转换
```
R5 = (R >> 3) & 0x1F  // 5位红色
G6 = (G >> 2) & 0x3F  // 6位绿色
B5 = (B >> 3) & 0x1F  // 5位蓝色
RGB565 = (R5 << 11) | (G6 << 5) | B5
```

### 字节序
大端序（Big-Endian）：高字节在前
```dart
rgb565[outIndex++] = (rgb565Value >> 8) & 0xFF;  // 高字节
rgb565[outIndex++] = rgb565Value & 0xFF;         // 低字节
```
