## 三、首次启动完整流程

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                         首次启动流程（只走一次）                                    │
└─────────────────────────────────────────────────────────────────────────────────┘

  ① APP启动             ② 同意协议            ③ 引导3页完成
     home                  pushReplacement       pushAndRemoveUntil

  ┌──────────┐         ┌──────────┐         ┌──────────┐
  │ Splash   │  ────▶  │Onboarding│  ────▶  │NoDevice  │
  └──────────┘         └──────────┘         └──────────┘
   栈: [Splash]         栈: [Onboarding]     栈: [NoDevice]
                        (替换了Splash)        (清空栈，重新开始)

   用户操作:             用户操作:             用户操作:
   点击"开始使用"        滑动3页后             到达 APP 主页 ✅
   → 弹出协议对话框      点击"开始探索"         后续启动直接到这里
   → 同意后跳转

  ┌─────────────────────────────────────────────────────────────────┐
  │ 关键点:                                                         │
  │ • Splash → Onboarding 用 pushReplacement（替换，不保留Splash）   │
  │ • Onboarding → NoDevice 用 pushAndRemoveUntil（清空整个栈）      │
  │ • 最终 NoDevice 是栈底唯一页面，返回 = 退出APP                    │
  └─────────────────────────────────────────────────────────────────┘
```


## 四、日常使用核心流程（非首次启动）

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                    日常使用流程：添加设备 → 扫描 → 连接 → 控制                      │
└─────────────────────────────────────────────────────────────────────────────────┘

  ① APP启动             ② 点击➕             ③ 扫描+连接成功        ④ 进入控制
     home                  push                pushReplace+push

  ┌──────────┐         ┌──────────┐         ┌──────────┐         ┌──────────┐
  │NoDevice  │  ────▶  │  Scan    │  ────▶  │ DevList  │  ────▶  │ Connect  │
  │ 添加设备  │  点击➕  │ 扫描设备  │  连接OK  │ 设备列表  │  自动    │ 主控制   │
  └──────────┘         ├──────────┤         ├──────────┤         ├──────────┤
   栈: [NoDevice]      │NoDevice  │         │NoDevice  │         │ DevList  │
                       └──────────┘         └──────────┘         ├──────────┤
                        栈: [NoDevice,       栈: [NoDevice,      │NoDevice  │
                              Scan]                DevList]       └──────────┘
                                                                  栈: [NoDevice,
                                                                        DevList,
                                                                        Connect]

  ┌─────────────────────────────────────────────────────────────────┐
  │ 步骤③详解:                                                      │
  │ 扫描连接成功后，点击"进入控制界面"按钮时:                           │
  │ 1. pushReplacement: Scan 被替换为 DevList                       │
  │    栈从 [NoDevice, Scan] 变为 [NoDevice, DevList]               │
  │ 2. push: 在 DevList 上面再推入 Connect                           │
  │    栈变为 [NoDevice, DevList, Connect]                           │
  └─────────────────────────────────────────────────────────────────┘
```


## 五、扫描失败/跳过流程

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                    扫描失败 / 未找到设备 / 点击跳过                                │
└─────────────────────────────────────────────────────────────────────────────────┘

  ① 正在扫描            ② 失败/跳过
                           pop

  ┌──────────┐         ┌──────────┐
  │  Scan    │  ────▶  │NoDevice  │  直接回到原来的页面
  ├──────────┤         └──────────┘
  │NoDevice  │          栈: [NoDevice]
  └──────────┘
   栈: [NoDevice,
         Scan]

  ┌─────────────────────────────────────────────────────────────────┐
  │ 触发条件（都是 pop）:                                            │
  │ • 蓝牙扫描完成但没找到设备                                       │
  │ • 连接设备失败                                                   │
  │ • 用户点击"跳过扫描"按钮                                         │
  │ • 扫描过程中发生异常                                              │
  └─────────────────────────────────────────────────────────────────┘
```


## 六、回退导航（返回键/返回按钮）

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                         完整回退链路                                              │
└─────────────────────────────────────────────────────────────────────────────────┘

  ┌──────────────────┐                                                
  │ DeviceConnect    │                                                
  │ 主控制页面       │                                                
  └────────┬─────────┘                                                
           │ 返回按钮 → pop                                           
           ▼                                                          
  ┌──────────────────┐                                                
  │ DeviceListScreen │                                                
  │ 我的设备页       │                                                
  └────────┬─────────┘                                                
           │ 返回按钮 → pop                                           
           ▼                                                          
  ┌──────────────────┐                                                
  │ NoDeviceScreen   │                                                
  │ 添加设备页 ➕    │                                                
  │ (APP主页/栈底)   │                                                
  └────────┬─────────┘                                                
           │ 返回键 → canPop=false → SystemNavigator.pop()            
           ▼                                                          
  ┌──────────────────┐                                                
  │    退出 APP      │                                                
  └──────────────────┘                                                

  回退总结: Connect ──pop──▶ DevList ──pop──▶ NoDevice ──退出──▶ APP关闭
           (3步退出，每步都有对应页面)
```

## 八、蓝牙断开场景

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                         蓝牙断开连接处理                                          │
└─────────────────────────────────────────────────────────────────────────────────┘

  ① 正在控制设备        ② 蓝牙断开 → 弹出对话框

  ┌──────────┐         ┌──────────────────────────────────┐
  │ Connect  │  ────▶  │  ┌────────────────────────────┐  │
  ├──────────┤         │  │    设备已断开               │  │
  │ DevList  │         │  │                            │  │
  ├──────────┤         │  │  [返回设备列表]  [重新连接]  │  │
  │NoDevice  │         │  └────────────────────────────┘  │
  └──────────┘         └──────────────────────────────────┘

  选择"返回设备列表":     选择"重新连接":
  → 关闭对话框            → 关闭对话框

  ┌──────────┐            → 失败: 弹出"连接失败"对话框
  │ DevList  │                    → pop (回到DevList)
  ├──────────┤

  └──────────┘
```

## 十二、栈累积防护验证

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                    反复操作不会栈累积的证明                                        │
└─────────────────────────────────────────────────────────────────────────────────┘

  场景: 用户反复 进入控制页 → 返回 → 再进入 → 再返回...

  第1轮:  [NoDevice] → push DevList → push Connect → pop → pop
          结果: [NoDevice]  ✅

  第2轮:  [NoDevice] → push Scan → 连接成功 → [NoDevice, DevList, Connect]
          → pop → [NoDevice, DevList] → pop → [NoDevice]
          结果: [NoDevice]  ✅

  第3轮:  [NoDevice] → push Scan → 失败 → pop → [NoDevice]
          结果: [NoDevice]  ✅

  从DevList反复进出:
  [NoDevice, DevList] → push Connect → pop → [NoDevice, DevList]
  [NoDevice, DevList] → push Connect → pop → [NoDevice, DevList]
  [NoDevice, DevList] → push Connect → pop → [NoDevice, DevList]
  永远是 [NoDevice, DevList]  ✅ 不会累积
```
