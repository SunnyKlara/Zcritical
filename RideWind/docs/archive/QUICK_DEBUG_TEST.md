# 快速调试测试指南

## 🎯 如何使用调试功能

### 1. 打开调试界面

1. 连接设备后,点击右上角的**菜单按钮**(三个点)
2. 选择 **"Logo 调试"** (橙色图标🐛)
3. 进入调试界面

### 2. 测试上传

1. **选择图片**: 点击蓝色的"选择图片"按钮
   - 从相册选择一张图片
   - 界面会自动显示预处理和压缩过程
   - 底部黑色区域会显示详细日志

2. **查看日志**: 
   ```
   [时间] 📷 开始选择图片...
   [时间] ✅ 图片已选择
   [时间] 🔄 开始预处理图片...
   [时间] ✅ 预处理完成
   [时间] 🗜️ 开始RLE压缩...
   [时间] ✅ 压缩完成
   ```

3. **上传**: 点击绿色的"上传"按钮
   - 查看实时上传进度
   - 查看每个步骤的日志

4. **复制日志**: 点击右上角的**复制图标**📋
   - 日志会复制到剪贴板
   - 粘贴给我分析

### 3. 日志颜色说明

- 🟢 **绿色**: 成功的操作
- 🔴 **红色**: 错误信息
- 🟡 **黄色**: 重要事件(开始/结束)
- 🔵 **蓝色**: 进度信息
- ⚪ **白色**: 普通信息

### 4. 关键信息

上传前会显示:
```
原始: 112.5 KB
压缩后: 54.5 KB  
压缩率: 48.4%
```

上传时会显示:
```
上传中: 25.0%
[进度条]
```

### 5. 硬件端日志

同时打开串口工具(波特率115200)查看硬件日志:
```
[LOGO] COMPRESSED START orig=115200 comp=55800 crc=305419896
[LOGO] Ready (compressed mode, 3488 packets)
[LOGO] Packet 100: 16 bytes decoded
[LOGO] Progress: 2% (seq=100/3488)
...
```

## 📋 需要提供的信息

当上传失败时,请提供:

### APP端信息
1. 点击右上角复制按钮
2. 粘贴完整日志

### 硬件端信息
1. 复制串口工具中的所有日志
2. 特别注意错误信息

### 图片信息
- 原始大小
- 压缩后大小
- 压缩率

## 🔍 常见问题

### Q: 看不到"Logo 调试"选项?
A: 
1. 确保已重新编译APP
2. 确保已连接设备
3. 点击右上角菜单按钮(三个点)

### Q: 日志区域是空的?
A: 
1. 先选择图片,日志会自动出现
2. 如果还是空的,说明代码有问题

### Q: 上传一直卡住?
A: 
1. 查看最后一条日志的时间
2. 查看硬件端串口日志
3. 复制所有日志给我

### Q: 显示"蓝牙未连接"?
A: 
1. 返回主界面
2. 确认设备已连接
3. 重新进入调试界面

## 💡 测试建议

1. **第一次测试**: 使用简单的纯色图片
   - 压缩率会很高(可能90%以上)
   - 传输速度快
   - 容易成功

2. **第二次测试**: 使用复杂图片
   - 压缩率较低(40-60%)
   - 传输时间长
   - 更能发现问题

3. **对比测试**: 
   - 先用"Logo 设置"上传(原版本)
   - 再用"Logo 调试"上传(调试版本)
   - 对比日志差异
