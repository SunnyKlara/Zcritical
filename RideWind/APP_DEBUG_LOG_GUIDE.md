# APP真机调试日志使用指南

## 功能说明

已在Logo上传界面添加**实时调试日志面板**，无需连接电脑即可查看详细的上传过程。

## 日志面板位置

## 日志颜色说明

- **绿色** ✅: 成功/确认信息
- **红色** ❌: 错误/失败信息
- **橙色** ⚠️: 警告信息
- **蓝色** 📤📥: 发送/接收数据
- **白色**: 普通信息

## 使用方法

### 1. 开始上传
1. 打开APP，连接蓝牙
2. 进入Logo设置界面
3. 选择图片并裁剪
4. 点击"上传(原版)"按钮
5. **日志面板会自动出现并显示实时日志**

### 2. 查看日志
- 日志会自动滚动到最新的一条
- 可以手动向上滚动查看历史日志
- 最多保留最近100条日志

### 3. 清空日志
- 点击日志面板右上角的 **[清空]** 按钮
- 或者重新开始上传会自动清空

### 4. 截图分享
- 当出现问题时，直接截图整个界面
- 日志面板会显示在截图中
- 发给开发者分析

## 常见问题的日志特征

### 问题1: 硬件无响应

```
00:35:14 📤 发送LOGO_START命令...
00:35:24 ⏰ 超时，尝试GET:LOGO...
00:35:29 📥 GET:LOGO响应: TIMEOUT
00:35:29 ❌ 上传失败: Exception: 硬件无响应，请重新编译固件
```

**解决**: 检查固件是否烧录最新版本

---

### 问题2: 卡在包9

```
00:35:21 📤 发送包 0/2965
00:35:27 ⏰ 超时！重试次数: 1/5
00:35:30 ⏰ 超时！重试次数: 2/5
00:35:33 ⏰ 超时！重试次数: 3/5
```

**解决**: 
1. 检查硬件端是否调用了`Logo_ProcessBuffer()`
2. 重新烧录固件

---

### 问题3: 频繁LOGO_BUSY

```
00:35:25 📤 发送包 100/2965
00:35:26 ⚠️ 硬件忙，等待200ms...
00:35:27 ✅ ACK确认: 109
00:35:30 📤 发送包 200/2965
00:35:31 ⚠️ 硬件忙，等待200ms...
```

**解决**: 硬件处理速度慢，属于正常现象，会自动重试

---

### 问题4: CRC校验失败

```
01:35:55 ✅ 传输完成，发送LOGO_END...
01:35:56 📥 LOGO_END响应: LOGO_FAIL:CRC:987654321
01:35:56 ❌ CRC校验失败: LOGO_FAIL:CRC:987654321
```

**解决**: 数据传输有误，重新上传

## 调试技巧

### 1. 对比正常日志
- 将你的日志与上面的"正常上传日志示例"对比
- 找出第一个不同的地方
- 那就是问题所在

### 2. 关注时间间隔
- 正常情况下，每100包大约需要2-3秒
- 如果某个包等待超过5秒，说明有问题

### 3. 查看ACK序号
- ACK序号应该是递增的: 9, 19, 29, 39...
- 如果ACK序号倒退或跳跃，说明有丢包

### 4. 截图完整界面
- 确保截图包含:
  - 进度条和百分比
  - 状态文本 (传输中 X/2965)
  - 硬件响应框
  - **完整的日志面板**

## 注意事项

1. **日志会占用一些内存**，如果APP运行缓慢，可以点击清空按钮
2. **日志只保留最近100条**，超过会自动删除最旧的
3. **日志会自动滚动**，但你可以手动向上查看历史
4. **日志面板高度固定200像素**，不会遮挡其他按钮

## 反馈问题时请提供

1. ✅ 完整的界面截图 (包含日志面板)
2. ✅ 问题描述 (卡在哪一步)
3. ✅ 固件版本和编译时间
4. ❌ 不需要连接电脑
5. ❌ 不需要adb命令
