# 🔧 开发者模式使用指南

## 功能说明

为了方便UI设计和调试，在**未连接设备页面**（No Device Screen）添加了一个开发者模式入口按钮。

## 使用方法

### 1. 进入未连接页面
- 启动应用后，如果没有扫描到设备，会自动进入未连接页面
- 或者在扫描页面点击"跳过扫描"按钮

### 2. 点击开发者模式按钮
- 位置：左下角
- 外观：橙色按钮，显示"DEV"图标和文字
- 样式：带有发光效果的圆角按钮

### 3. 直接进入控制页面
- 点击后会创建一个虚拟设备（Mock Device）
- 自动跳转到设备控制页面（Device Connect Screen）
- 可以直接进行UI设计和调试，无需真实蓝牙连接

## 虚拟设备信息

```dart
DeviceModel(
  id: 'dev-mock-001',
  name: 'DEV Mock Device',
  rssi: -50,
  isConnected: true,
)
```

## 按钮样式

- **位置**: 左下角（bottom: 40px, left: 20px）
- **尺寸**: 120px × 48px
- **颜色**: 橙色（#FF9800）
- **图标**: developer_mode
- **文字**: "DEV"

## 适用场景

✅ UI设计和布局调整  
✅ 交互逻辑测试  
✅ 动画效果调试  
✅ 响应式布局验证  
✅ 颜色和样式调整  

❌ 不适用于蓝牙通信测试（需要真实设备）

## 注意事项

1. **仅用于开发调试**：此功能仅用于UI开发，不会建立真实的蓝牙连接
2. **蓝牙命令无效**：在开发者模式下，所有蓝牙命令都不会真正发送到硬件
3. **状态模拟**：设备状态变化仅在本地模拟，不会同步到真实硬件
4. **生产环境**：发布正式版本前可以选择隐藏或移除此按钮

## 如何隐藏/移除开发者模式按钮

### 方法1: 条件编译（推荐）
```dart
// 在 no_device_screen.dart 顶部添加
const bool kEnableDevMode = true; // 开发环境设为 true，生产环境设为 false

// 在按钮代码外层添加条件
if (kEnableDevMode)
  Positioned(
    // ... 开发者按钮代码
  ),
```

### 方法2: 完全移除
直接删除 `no_device_screen.dart` 中的开发者模式按钮代码块（第220-270行左右）
