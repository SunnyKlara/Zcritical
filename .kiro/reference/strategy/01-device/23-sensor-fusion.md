# 23. 传感器融合

## 是什么

将多个传感器的原始数据通过算法融合，得到比单一传感器更准确、更鲁棒的状态估计。典型场景：IMU（加速度计+陀螺仪+磁力计）融合得到姿态角，或多源速度估计。

## 为什么需要

- 单一传感器有噪声、漂移、盲区
- 加速度计：静态准但动态抖
- 陀螺仪：短期准但长期漂移
- 磁力计：绝对方向但易受干扰
- 融合后：取各传感器优势，互补缺陷
- 产品应用：骑行姿态检测、摔车检测、精确速度估计、手势识别

## 技术架构

## 技术栈选型

| 组件 | 技术 | 说明 |
|------|------|------|
| IMU 芯片 | MPU6050 / ICM-42688 / BMI270 | 6轴/9轴 |
| 通信接口 | I2C (400kHz) 或 SPI (10MHz) | SPI 更快但占引脚 |
| 融合算法 | Madgwick AHRS | 计算量小，适合 MCU |
| 备选算法 | 互补滤波（最简单）/ EKF（最精确） | 按精度需求选 |
| 采样率 | 100-1000Hz | 骑行场景 200Hz 足够 |
| 数学库 | 定点数或 ESP32 硬件 FPU | S3 有 FPU，直接用 float |

## 实现步骤

### Phase 1：硬件驱动（2h）

1. **IMU 驱动编写**
   ```c
   // drivers/drv_imu.c
   typedef struct {
       float acc[3];   // m/s²
       float gyro[3];  // rad/s
       float mag[3];   // uT (如果有)
   } imu_raw_t;
   
   esp_err_t drv_imu_init(void);
   esp_err_t drv_imu_read(imu_raw_t *data);
   ```

2. **校准流程**
   - 陀螺仪零偏：静止时采集 1000 个样本取平均
   - 加速度计：六面校准法（每面朝上静置）
   - 磁力计：画 8 字校准（椭球拟合）

### Phase 2：融合算法实现（3-4h）

3. **Madgwick 滤波器**
   ```c
   // services/ahrs.c
   typedef struct {
       float q[4];     // 四元数 [w, x, y, z]
       float beta;     // 算法增益（0.01-0.1）
       float dt;       // 采样间隔
   } madgwick_t;
   
   void madgwick_update(madgwick_t *f, 
                        float gx, float gy, float gz,
                        float ax, float ay, float az);
   void madgwick_get_euler(madgwick_t *f, 
                           float *roll, float *pitch, float *yaw);
   ```

4. **互补滤波（简化版）**
   ```c
   // 最简单的融合：98% 陀螺仪积分 + 2% 加速度计修正
   angle = 0.98 * (angle + gyro * dt) + 0.02 * acc_angle;
   ```

### Phase 3：应用场景（2-3h）

5. **摔车检测**
   - 加速度矢量模 > 3g 且持续 > 50ms → 疑似摔车
   - 随后 2 秒内姿态角变化 > 60° → 确认摔车
   - 触发：BLE 通知 App → App 弹窗确认 → 超时未确认则发送 SOS

6. **速度校正**
   - 加速度计积分得到速度增量
   - 与风速传感器/GPS 融合，修正漂移
   - 卡尔曼滤波：预测（IMU）+ 更新（外部传感器）

7. **倾斜角显示**
   - 实时显示骑行倾斜角（过弯时）
   - LCD 或 App 显示，增加趣味性

### Phase 4：实时任务集成（1h）

8. **FreeRTOS 任务设计**
   ```c
   // 200Hz 采样任务，优先级高于 UI
   void imu_task(void *arg) {
       TickType_t last_wake = xTaskGetTickCount();
       while (1) {
           drv_imu_read(&raw);
           madgwick_update(&ahrs, ...);
           // 检测事件
           check_crash_detection(&ahrs);
           vTaskDelayUntil(&last_wake, pdMS_TO_TICKS(5)); // 200Hz
       }
   }
   ```

## 关键坑点

| 坑 | 后果 | 解法 |
|----|------|------|
| 陀螺仪温漂 | 静止时角度缓慢偏移 | 开机校准 + 运行时零偏估计 |
| 磁场干扰 | 电机/喇叭磁铁影响磁力计 | 硬铁/软铁补偿，或不用磁力计 |
| 振动噪声 | 骑行振动淹没有用信号 | 低通滤波 + 提高 beta 值 |
| 采样率不稳 | 融合算法假设固定 dt | 用实际时间戳计算 dt |
| 万向锁 | 欧拉角在 ±90° pitch 时奇异 | 内部用四元数，显示时才转欧拉角 |
| CPU 占用 | 1000Hz 采样 + 浮点运算 | ESP32-S3 FPU 够用，200Hz 足矣 |

## 与 RideWind 的关系

- 当前状态：无 IMU 硬件（PCB 未预留）
- 优先级：P5（下一代硬件）
- 价值：摔车检测是差异化功能，倾斜角显示增加趣味
- 前置条件：下一版 PCB 加 IMU 芯片（成本 +3-5 元）
