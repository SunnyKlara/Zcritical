# 38. App 国际化（i18n）

## 是什么
让 App 支持多语言。用户可以切换语言，或自动跟随系统语言。包括文本翻译、日期/数字格式、RTL 布局、图片/图标本地化。

## 为什么需要
- 产品要卖到海外（东南亚、欧美）
- App Store 上架不同地区需要对应语言
- 英文界面 = 国际化形象 = 更高定价空间
- Flutter 原生支持 i18n，成本不高但价值大
- 越早做越容易（后期补翻译要改几百个文件）

## 技术架构
```
┌─────────────────────────────────────────────────┐
│              国际化架构                           │
├─────────────────────────────────────────────────┤
│                                                  │
│  [翻译文件]                                      │
│   lib/l10n/                                     │
│   ├─ app_zh.arb    (中文，基准)                 │
│   ├─ app_en.arb    (英文)                       │
│   ├─ app_ja.arb    (日文)                       │
│   └─ app_ko.arb    (韩文)                       │
│                                                  │
│  [生成层]                                        │
│   flutter gen-l10n → AppLocalizations 类        │
│   类型安全，编译时检查缺失翻译                  │
│                                                  │
│  [使用层]                                        │
│   AppLocalizations.of(context).welcomeMessage   │
│   自动跟随系统语言 / 手动切换                   │
│                                                  │
│  [格式化]                                        │
│   intl 包：日期、数字、货币按 locale 格式化     │
│                                                  │
└─────────────────────────────────────────────────┘
```

## 技术栈选型
| 组件 | 技术 | 说明 |
|------|------|------|
| 框架 | Flutter intl + gen-l10n | 官方方案，类型安全 |
| 翻译文件 | ARB (Application Resource Bundle) | Flutter 标准格式 |
| 复数/性别 | ICU MessageFormat | 处理复杂语法 |
| 翻译管理 | Crowdin / Lokalise | 协作翻译平台（可选） |
| RTL | Directionality widget | 阿拉伯语/希伯来语 |

## 实现步骤

### Phase 1：基础设施（1-2h）
1. pubspec.yaml 启用 generate: true
2. 创建 l10n.yaml 配置文件
3. 创建基准 ARB 文件（中文）
4. MaterialApp 配置 localizationsDelegates

### Phase 2：文本提取（2-3h）
5. 遍历所有硬编码中文字符串 → 替换为 AppLocalizations 引用
6. 处理带参数的字符串：`"速度: {speed} km/h"` → `speed(speed)`
7. 处理复数：`"{count} 台设备"` → `deviceCount(count)`

### Phase 3：翻译（2-3h）
8. 英文翻译（自己或 AI 辅助）
9. 日文/韩文（如果目标市场需要）
10. 校对：确保翻译在 UI 中不溢出（德语/俄语通常比英语长 30%）

### Phase 4：格式化（1h）
11. 日期格式：中文 "2026年5月18日" / 英文 "May 18, 2026"
12. 数字格式：中文 "1,234" / 德文 "1.234"
13. 货币：¥ / $ / €

## 关键坑点
| 坑 | 后果 | 解法 |
|----|------|------|
| 文本溢出 | 翻译后文字太长撑破布局 | 用 Flexible/Expanded，测试长文本 |
| 硬编码遗漏 | 部分文字没翻译 | lint 规则检查硬编码字符串 |
| 图片中的文字 | 图片里的中文没法翻译 | 文字和图片分离，或准备多语言图片 |
| 复数规则 | 英语 1/other，中文无复数 | ICU MessageFormat 处理 |
| 上下文缺失 | 翻译者不知道文字出现在哪 | ARB 文件加 @description 注释 |
| 动态内容 | 后端返回的文字没翻译 | 后端也做 i18n，或前端映射 |
