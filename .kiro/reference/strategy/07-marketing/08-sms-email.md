# 8. 短信/邮件触达

## 是什么
通过短信和邮件触达用户的完整系统。短信用于验证码、重要通知（订单/物流）；邮件用于营销活动、产品更新、用户召回。是推送通知之外的补充触达通道。

## 为什么需要
- 注册/登录验证码：短信是国内最通用的身份验证方式
- 重要通知不能只靠推送：用户可能关闭了通知权限
- 营销邮件：成本极低（几乎免费），适合内容丰富的通知
- 用户召回：30 天未活跃 → 短信/邮件提醒
- 订单通知：发货、签收、售后进度

## 技术架构
```
┌─────────────────────────────────────────────────────┐
│              短信/邮件触达架构                        │
├─────────────────────────────────────────────────────┤
│                                                      │
│  [触发源]                                            │
│   业务事件 → 验证码/通知（实时）                   │
│   运营后台 → 营销活动（批量/定时）                 │
│   自动化规则 → 用户行为触发                        │
│       │                                              │
│       ▼                                              │
│  [消息服务]                                          │
│   模板管理 → 变量替换 → 频率控制 → 发送队列      │
│       │                                              │
│       ├─── [短信通道]                               │
│       │     阿里云 SMS / 腾讯云 SMS                 │
│       │     模板审核 → 签名 → 发送 → 状态回调     │
│       │                                              │
│       └─── [邮件通道]                               │
│             SendGrid / 阿里云邮件 / AWS SES         │
│             HTML 模板 → 发送 → 打开/点击追踪      │
│       │                                              │
│       ▼                                              │
│  [数据层]                                            │
│   发送记录 / 送达状态 / 退订管理 / 成本统计       │
│                                                      │
└─────────────────────────────────────────────────────┘
```

## 技术栈选型
| 组件 | 技术 | 说明 |
|------|------|------|
| 短信（国内） | 阿里云 SMS | 到达率高，模板审核快 |
| 短信（国际） | Twilio | 全球覆盖 |
| 邮件（事务） | SendGrid / AWS SES | 验证码/通知类 |
| 邮件（营销） | SendGrid / Mailchimp | 批量发送 + 数据分析 |
| 模板引擎 | Handlebars / Mustache | 变量替换 |
| 队列 | Redis Streams / RabbitMQ | 异步发送，削峰 |
| 退订管理 | 自建 + 一键退订链接 | 合规要求 |

## 实现步骤

### Phase 1：短信验证码（2h）
1. 阿里云 SMS 开通 + 签名申请 + 模板审核
2. 验证码发送逻辑：
   ```
   POST /api/sms/send-code
   {
     "phone": "13800138000",
     "scene": "register"  // register/login/reset_password
   }
   ```
3. 验证码生成：6 位随机数 → Redis 存储（TTL=5min）
4. 频率限制：
   - 同一手机号：60 秒内不能重发
   - 同一 IP：每小时最多 10 次
   - 同一手机号：每天最多 5 次

### Phase 2：短信通知（1-2h）
5. 通知模板：
   - 订单发货：「您的 RideWind 已发货，快递单号 ${tracking_no}」
   - 设备异常：「您的设备 ${device_name} 检测到异常，请打开 App 查看」
6. 批量发送：运营后台选择用户群 → 异步队列发送
7. 发送状态回调：阿里云回调 → 更新发送记录（成功/失败/用户关机）

### Phase 3：邮件系统（2-3h）
8. SendGrid 集成 + 域名验证（SPF/DKIM/DMARC）
9. 邮件模板设计：
   - 事务邮件：验证码、密码重置（简洁）
   - 通知邮件：OTA 更新、订单确认（品牌化）
   - 营销邮件：新功能介绍、活动促销（富媒体）
10. HTML 邮件模板：响应式设计，兼容各邮件客户端
11. 变量替换：`{{username}}`、`{{device_name}}`、`{{action_url}}`

### Phase 4：退订与合规（1h）
12. 一键退订：每封营销邮件底部必须有退订链接
13. 退订管理：用户退订后标记，不再发送营销类
14. 事务类不受退订影响（验证码、安全通知仍然发送）
15. 隐私合规：GDPR 要求明确同意才能发营销邮件

### Phase 5：数据分析与优化（1h）
16. 短信：发送量/送达率/验证码使用率/成本
17. 邮件：发送量/打开率/点击率/退订率/退信率
18. 优化：
    - 发送时间优化（工作日 10:00-11:00 打开率最高）
    - 标题 A/B 测试
    - 内容个性化（按用户行为推荐）

## 关键坑点
| 坑 | 后果 | 解法 |
|----|------|------|
| 短信被刷 | 恶意调用验证码接口烧钱 | 图形验证码前置 + IP 限流 + 手机号黑名单 |
| 邮件进垃圾箱 | 用户看不到 | SPF/DKIM/DMARC 配置 + 预热 IP |
| 模板审核不过 | 短信发不出去 | 提前了解审核规则，避免敏感词 |
| 退订率高 | 发送信誉下降 | 控制频率 + 内容有价值 + 精准分群 |
| 成本失控 | 短信费用暴涨 | 频率限制 + 优先用免费通道（推送/邮件） |
| 国际短信贵 | 海外用户验证码成本高 | 海外优先用邮件验证 |

## 与 RideWind 的关系
- 当前状态：无短信/邮件能力
- 优先级：P4（用户系统做完后，验证码是刚需）
- 起步：阿里云 SMS（验证码）+ SendGrid（邮件通知）
- 成本：短信 ¥0.045/条，邮件 SendGrid 免费 100 条/天
- 依赖：#2 用户系统（有手机号/邮箱才能发）
