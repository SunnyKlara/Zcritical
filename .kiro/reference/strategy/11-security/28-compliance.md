# 28. 安全合规

## 是什么
确保产品符合法律法规和行业标准：数据隐私（GDPR/个保法）、无线认证（CE/FCC）、App 上架合规。是产品合法销售的前提条件，不合规 = 无法上架/被罚款/被起诉。

## 为什么需要
- 不合规 = 无法上架：App Store/Google Play 有严格审核规则
- 不合规 = 被罚款：GDPR 罚款最高全球营收 4%，个保法最高 5000 万
- 出海必须过认证：CE（欧洲）/FCC（美国）无线认证是硬性要求
- 用户信任基础：隐私政策透明 = 用户愿意使用
- 法律风险：数据泄露/未授权收集 → 用户起诉

| 组件 | 技术 | 说明 |

## 实现步骤

### Phase 1：隐私政策与用户同意（2-3h）
1. 隐私政策文档编写：
   ```
   必须包含：
   - 收集哪些数据（设备信息/使用数据/位置）
   - 为什么收集（功能必需/改进产品/安全）
   - 如何存储和保护
   - 是否分享给第三方
   - 用户权利（访问/修改/删除/撤回同意）
   - 数据保留期限
   - 联系方式
   ```
2. 用户同意流程：
   - 首次启动 App → 隐私政策弹窗
   - 用户阅读并同意 → 记录同意时间和版本
   - 拒绝 → 只能使用基础功能（无数据采集）
3. 数据分类：必要数据/可选数据/敏感数据

### Phase 2：数据安全技术措施（3-4h）
4. 传输加密：HTTPS (TLS 1.3) + Certificate Pinning
5. 存储加密：
   - 服务端：数据库字段级加密
   - App 端：Secure Storage（Keychain/Keystore）
   - 设备端：NVS 加密分区
6. 权限最小化：
   ```
   必需权限：蓝牙（连接设备）、位置（BLE 扫描 Android 必需）
   按需权限：相机（扫码）、存储（导出）、通知（推送）
   不申请：通讯录/短信/电话/麦克风
   ```
7. 数据删除：用户注销 → 30 天内删除所有个人数据

### Phase 3：无线认证准备（2-3h）
8. CE 认证（欧洲市场）：
   - RED 2014/53/EU + EMC 2014/30/EU
   - 测试：射频/EMC/安全
   - 周期：4-6 周，费用：¥8000-15000
9. FCC 认证（美国市场）：
   - Part 15 Subpart C
   - 周期：3-4 周，费用：¥6000-12000
10. 蓝牙 SIG 认证：
    - 所有使用蓝牙商标的产品必须认证
    - ESP32 模组可能已有认证，产品端只需 End Product Listing

### Phase 4：App 上架合规（2-3h）
11. Apple App Store：
    - 隐私政策 URL + 隐私标签 + ATT 弹窗
    - 审核指南遵守（4.2 最小功能/5.1 隐私）
12. Google Play：
    - 数据安全声明 + 权限说明 + 目标 API 级别
13. 国内应用商店：
    - App 备案 + 隐私弹窗 + 权限说明 + SDK 清单

### Phase 5：持续合规管理（1-2h）
14. 安全审计：定期代码扫描 + 依赖漏洞监控
15. 合规检查清单：每次发版前检查
16. 事件响应：数据泄露 72 小时内通知（GDPR 要求）

## 关键坑点
| 坑 | 后果 | 解法 |
|----|------|------|
| 隐私政策模板化 | 不符合实际被投诉 | 根据实际数据收集定制 |
| 认证周期长 | 延误上市时间 | 提前 2 个月开始 |
| 认证整改 | 测试不通过需改硬件 | 设计阶段考虑 EMC |
| 权限过度申请 | 审核被拒 | 只申请必要权限 |
| 第三方 SDK 合规 | SDK 偷偷收集数据 | 审查每个 SDK 隐私政策 |
| 法规更新 | 新法规出台不知道 | 关注行业动态，定期审查 |
