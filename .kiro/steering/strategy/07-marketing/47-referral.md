# 47. 裂变/邀请系统

## 是什么
老用户邀请新用户注册或购买产品，双方获得奖励（优惠券/积分/会员天数），通过社交关系链实现低成本用户增长。是"让用户帮你拉用户"的增长引擎。

## 为什么需要
- 获客成本最低：老用户推荐的转化率是广告的 3-5 倍
- 信任背书：朋友推荐比广告更可信
- 社交裂变：一个用户带来 N 个用户，指数增长
- 用户质量高：被推荐来的用户留存率通常更高
- 口碑传播：好产品自带传播属性，系统化放大

## 技术架构
```
┌─────────────────────────────────────────────────────┐
│              裂变/邀请系统架构                        │
├─────────────────────────────────────────────────────┤
│                                                      │
│  [邀请生成]                                          │
│   用户 A → 生成专属邀请码/链接/二维码              │
│       │                                              │
│       ▼                                              │
│  [分享渠道]                                          │
│   微信分享 / 朋友圈 / 短信 / 复制链接             │
│   小程序卡片 / App 内分享                          │
│       │                                              │
│       ▼                                              │
│  [新用户注册]                                        │
│   用户 B 通过邀请链接注册                          │
│   → 绑定邀请关系 (A 邀请了 B)                     │
│       │                                              │
│       ▼                                              │
│  [奖励发放]                                          │
│   条件满足 → 双方发放奖励                          │
│   A: 邀请奖励（积分/优惠券/会员天数）             │
│   B: 新人奖励（首单折扣/体验会员）                │
│       │                                              │
│       ▼                                              │
│  [防作弊]                                            │
│   设备指纹 / IP 检测 / 行为分析 / 人工审核        │
│                                                      │
└─────────────────────────────────────────────────────┘
```

## 技术栈选型
| 组件 | 技术 | 说明 |
|------|------|------|
| 邀请码 | 短码生成（6-8位字母数字） | 易记易传播 |
| 邀请链接 | 短链接 + 深度链接 | 追踪来源 + 直达 App |
| 二维码 | qr_flutter / 后端生成 | 线下场景 |
| 分享 SDK | 微信 SDK / share_plus | 社交平台分享 |
| 关系存储 | PostgreSQL (邀请关系表) | 树形结构 |
| 防作弊 | 设备指纹 + 规则引擎 | 识别刷单 |
| 奖励发放 | 异步队列 + 幂等 | 确保不重复发放 |

## 实现步骤

### Phase 1：邀请码系统（2h）
1. 邀请码生成：
   ```sql
   CREATE TABLE invite_codes (
     code        VARCHAR(8) PRIMARY KEY,
     user_id     UUID REFERENCES users(id),
     created_at  TIMESTAMPTZ DEFAULT NOW(),
     used_count  INT DEFAULT 0,
     max_uses    INT DEFAULT 100
   );
   ```
2. 邀请关系表：
   ```sql
   CREATE TABLE invite_relations (
     id          UUID PRIMARY KEY,
     inviter_id  UUID REFERENCES users(id),
     invitee_id  UUID REFERENCES users(id),
     invite_code VARCHAR(8),
     channel     VARCHAR(50),  -- wechat/sms/link/qrcode
     status      VARCHAR(20),  -- registered/qualified/rewarded
     created_at  TIMESTAMPTZ DEFAULT NOW()
   );
   ```
3. 邀请码 API：`GET /api/invite/my-code` → 返回用户专属邀请码

### Phase 2：分享与追踪（2-3h）
4. 邀请链接生成：`https://ridewind.app/invite/{code}`
5. 短链接服务：长链接 → 短链接（追踪点击量）
6. 深度链接：
   - 已安装 App → 直接打开 App 并填入邀请码
   - 未安装 → 跳转应用商店，安装后首次打开自动填入
7. 微信小程序分享卡片：自定义标题/图片/路径
8. 分享渠道追踪：记录每次分享的渠道来源

### Phase 3：奖励机制（2h）
9. 奖励规则配置：
   ```json
   {
     "inviter_reward": {
       "type": "points",
       "amount": 100,
       "condition": "invitee_bindDevice"
     },
     "invitee_reward": {
       "type": "coupon",
       "amount": 20,
       "condition": "register"
     }
   }
   ```
10. 奖励发放流程：
    - 新用户注册 → 绑定邀请关系 → 发放新人奖励
    - 新用户完成关键行为（绑定设备）→ 发放邀请人奖励
11. 奖励上限：每人每月最多邀请 N 人获得奖励
12. 阶梯奖励：邀请 3 人/10 人/50 人 → 不同等级奖励

### Phase 4：防作弊（2h）
13. 设备指纹检测：同一设备多次注册 → 拒绝
14. IP 聚集检测：短时间内同一 IP 大量注册 → 标记可疑
15. 行为分析：注册后无任何真实使用行为 → 不发放奖励
16. 规则引擎：
    - 同一 WiFi 下 24h 内注册 >3 人 → 人工审核
    - 邀请人和被邀请人设备指纹相似度 >80% → 拒绝
17. 人工审核队列：可疑记录人工确认后再发放

### Phase 5：数据分析（1h）
18. 核心指标：
    - 邀请转化率：分享次数 → 注册数 → 活跃数
    - 病毒系数 K：每个用户平均带来多少新用户
    - 渠道效果：微信/短信/链接哪个转化率高
19. 邀请排行榜：激励头部用户持续邀请
20. ROI 分析：每个新用户的获客成本 vs 广告获客成本

## 关键坑点
| 坑 | 后果 | 解法 |
|----|------|------|
| 羊毛党 | 虚假注册刷奖励 | 设备指纹 + 行为验证 + 延迟发放 |
| 深度链接失败 | 邀请码丢失 | 多重回退（剪贴板/手动输入） |
| 奖励重复发放 | 多发钱 | 幂等设计 + 唯一约束 |
| 微信封链接 | 分享链接被屏蔽 | 用小程序卡片替代 H5 链接 |
| 激励不足 | 用户没动力分享 | 测试不同奖励力度，找到最优 |
| 关系链复杂 | 多级邀请计算困难 | 只做一级邀请（A→B），不做多级 |

## 与 RideWind 的关系
- 当前状态：无裂变/邀请系统
- 优先级：P5（有用户系统 + 一定用户基数后）
- 依赖：#2 用户系统 + #19 微信小程序（分享载体）
- 起步场景：邀请好友注册 → 双方获得高级音效包体验
- 目标：病毒系数 K > 0.3（每 10 个用户带来 3 个新用户）
