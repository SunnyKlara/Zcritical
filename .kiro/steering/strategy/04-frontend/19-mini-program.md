# 19. 微信小程序

## 是什么
微信生态内的轻量级应用。用户无需下载 App，扫码即用。适合轻度交互场景：设备配网、快速查看状态、分享、售后服务。

## 为什么需要
- 降低用户门槛：不用下载 App 就能体验基础功能
- 获客渠道：微信社交裂变（分享小程序卡片）
- 售后入口：扫设备二维码 → 小程序 → 查保修/提工单
- 配网工具：WiFi 设备配网（未来扩展）
- 国内用户习惯：很多人不愿装 App，但愿意用小程序

## 技术架构
```
┌─────────────────────────────────────────────────┐
│              小程序架构                           │
├─────────────────────────────────────────────────┤
│                                                  │
│  [小程序前端]                                    │
│   Taro / uni-app / 原生 WXML                   │
│   ├─ 首页：产品介绍 + 快速入口                 │
│   ├─ 设备绑定：扫码 → 绑定 → 基础控制         │
│   ├─ 售后：保修查询 / 提交工单 / FAQ           │
│   ├─ 社区：用户分享 / 配置方案                 │
│   └─ 个人中心：订单 / 设备列表 / 设置         │
│       │                                          │
│       ▼                                          │
│  [后端 API]（复用主后端）                       │
│   微信登录 → openid → 关联用户                 │
│                                                  │
│  [微信能力]                                      │
│   BLE API / 扫码 / 支付 / 订阅消息 / 分享     │
│                                                  │
└─────────────────────────────────────────────────┘
```

## 技术栈选型
| 组件 | 技术 | 理由 |
|------|------|------|
| 框架 | Taro 3 (React 语法) | 跨端（小程序+H5），React 生态复用 |
| UI 库 | NutUI / Taro UI | 适配小程序的组件库 |
| 状态管理 | Zustand / MobX | 轻量 |
| BLE | wx.createBLEConnection | 微信原生 BLE API |
| 登录 | wx.login → 后端换 session | 微信静默登录 |
| 部署 | 微信开发者工具上传 | 审核后发布 |

## 实现步骤

### Phase 1：项目搭建（1-2h）
1. Taro 3 + React + TypeScript 初始化
2. 微信登录流程（wx.login → code → 后端换 token）
3. 请求封装 + token 管理

### Phase 2：核心页面（3-4h）
4. 首页：产品展示 + 功能入口
5. 设备绑定：扫码获取 SN → 调用绑定 API
6. 基础控制：亮度/颜色/模式（通过后端 → App → BLE 转发）
7. 个人中心：设备列表 / 订单 / 设置

### Phase 3：售后模块（2h）
8. 保修查询：输入 SN → 查询保修状态
9. 提交工单：问题描述 + 图片上传
10. FAQ：常见问题列表

### Phase 4：社交裂变（1-2h）
11. 分享卡片：自定义分享图片和标题
12. 邀请有礼：生成专属邀请码
13. 订阅消息：OTA 更新通知、工单回复通知

## 关键坑点
| 坑 | 后果 | 解法 |
|----|------|------|
| BLE 兼容性 | 不同手机 BLE 行为不同 | 充分测试 + 错误重试 |
| 审核被拒 | 功能不完整/违规 | 提前了解审核规则，准备测试账号 |
| 包大小限制 | 主包 2MB 限制 | 分包加载 + 图片走 CDN |
| 登录态过期 | 用户操作中断 | 静默续期 + 引导重新登录 |
| 与 App 功能重复 | 维护两套代码 | 小程序只做轻量功能，重度操作引导下载 App |

## 与 RideWind 的关系
- 当前状态：无小程序
- 优先级：P4（App 稳定后扩展）
- 定位：轻量入口（售后/分享/配网），不替代 App
- 依赖：后端 API 就绪
