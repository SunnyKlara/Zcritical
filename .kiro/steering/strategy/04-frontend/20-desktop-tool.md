# 20. 桌面配置工具

## 是什么
PC/Mac 端的桌面应用程序。用于固件烧录、设备配置、日志查看、音频素材管理等"重度操作"。面向开发者和高级用户。

## 为什么需要
- 固件烧录需要 USB 串口连接（手机做不了）
- 大文件传输（音频素材、Logo 批量导入）USB 比 BLE 快 100 倍
- 开发调试：实时查看串口日志、修改配置参数
- 产线工具：批量烧录 + 产测（#45）的 PC 端界面
- 高级用户：自定义音频编辑、灯效编程

## 技术架构
```
┌─────────────────────────────────────────────────┐
│              桌面配置工具                         │
├─────────────────────────────────────────────────┤
│                                                  │
│  [UI 层]                                         │
│   Electron + React / Tauri + Vue               │
│   ├─ 设备连接（USB 串口选择）                  │
│   ├─ 固件烧录（esptool 集成）                  │
│   ├─ 实时日志（串口监视器）                    │
│   ├─ 配置编辑（参数表格 + 一键写入）          │
│   ├─ 音频管理（导入/预览/转换/上传）          │
│   └─ 产测面板（自动化测试 + 报告）            │
│       │                                          │
│       ▼                                          │
│  [通信层]                                        │
│   serialport (Node.js) / Tauri serial plugin    │
│   USB CDC → 串口协议 → 设备                    │
│                                                  │
│  [工具层]                                        │
│   esptool.py（烧录）/ ffmpeg（音频转换）       │
│   Python 脚本调用 / 内置二进制                  │
│                                                  │
└─────────────────────────────────────────────────┘
```

## 技术栈选型
| 组件 | 技术 | 理由 |
|------|------|------|
| 框架 | Tauri 2.0 (Rust + Web) | 比 Electron 小 10 倍，性能好 |
| 备选 | Electron + React | 生态更成熟，串口库更稳定 |
| UI | React + Ant Design | 复用 Web 管理后台组件 |
| 串口 | serialport (Node) / tauri-plugin-serial | 跨平台串口通信 |
| 烧录 | esptool.py 内置 | ESP32 官方烧录工具 |
| 音频 | ffmpeg + Web Audio API | 格式转换 + 波形预览 |
| 打包 | Tauri bundler / electron-builder | 生成安装包 |

## 实现步骤

### Phase 1：项目搭建（2h）
1. Tauri 2.0 + React + TypeScript 初始化
2. 串口通信插件集成
3. 基础 UI 布局（侧边栏导航 + 内容区）

### Phase 2：串口通信（2-3h）
4. 串口扫描 + 自动识别 ESP32 设备
5. 串口监视器（实时日志显示，支持过滤/搜索）
6. 命令发送（和 BLE 协议复用同一套文本协议）

### Phase 3：固件烧录（2-3h）
7. 集成 esptool.py（Python 环境或预编译二进制）
8. 一键烧录：选择 bin 文件 → 自动检测端口 → 烧录 → 进度条
9. 分区烧录：可选只烧 app / storage / 全量

### Phase 4：配置与音频（2-3h）
10. 配置编辑器：读取设备当前配置 → 表格编辑 → 一键写入
11. 音频管理：导入 MP3/WAV → 转换为设备格式 → 预览 → 上传
12. Logo 管理：导入图片 → 缩放裁剪 → 转 RGB565 → 上传

### Phase 5：产测集成（1-2h）
13. 产测面板：连接设备 → 自动运行测试 → 显示结果 → 生成报告
14. 批量模式：一台 PC 连多台设备（USB Hub），并行测试

## 关键坑点
| 坑 | 后果 | 解法 |
|----|------|------|
| 串口驱动 | Windows 需要 CH340/CP2102 驱动 | 安装包内置驱动 / 提示安装 |
| 跨平台差异 | Mac/Win/Linux 串口路径不同 | 抽象层统一处理 |
| esptool 依赖 | 需要 Python 环境 | 打包预编译 esptool 二进制 |
| 大文件传输 | 串口速率限制 | 921600 baud + 进度显示 |
| 权限问题 | Linux 串口需要 dialout 组 | 安装时自动配置 / 提示 |
| 应用签名 | Mac 未签名应用被拒绝运行 | Apple Developer 签名 |
