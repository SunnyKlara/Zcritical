# 39. App 无障碍（Accessibility）

## 是什么
让视障、听障、运动障碍用户也能使用 App。包括屏幕阅读器支持（TalkBack/VoiceOver）、大字体适配、高对比度、触摸目标大小、键盘导航。

## 为什么需要
- 法律要求：欧美市场 ADA/EAA 合规，不达标可能被诉
- App Store 审核：Apple 重视无障碍，影响推荐权重
- 用户群扩大：全球 10 亿+ 残障人士
- 产品质量：无障碍做得好 = 所有用户体验都好（大字体、高对比度对老年人也友好）
- 技术素养：面试加分项，体现工程师的专业度

## 技术架构
```
┌─────────────────────────────────────────────────┐
│              无障碍架构                           │
├─────────────────────────────────────────────────┤
│                                                  │
│  [语义层] Semantics Widget                      │
│   每个可交互元素都有语义标签                    │
│   label / hint / value / onTap                  │
│                                                  │
│  [屏幕阅读器]                                    │
│   iOS: VoiceOver                                │
│   Android: TalkBack                             │
│   读出语义标签 → 用户听到 → 手势操作           │
│                                                  │
│  [视觉适配]                                      │
│   大字体：MediaQuery.textScaleFactor            │
│   高对比度：暗色/亮色主题 + 对比度 ≥ 4.5:1     │
│   色盲友好：不仅靠颜色区分状态                  │
│                                                  │
│  [交互适配]                                      │
│   触摸目标 ≥ 48x48dp                           │
│   焦点顺序合理（Tab 导航）                     │
│   动画可关闭（前庭障碍）                       │
│                                                  │
└─────────────────────────────────────────────────┘
```

## 技术栈选型
| 组件 | 技术 | 说明 |
|------|------|------|
| 语义标注 | Flutter Semantics widget | 原生支持 |
| 测试 | Accessibility Inspector (Xcode) | iOS 无障碍检查 |
| 测试 | Accessibility Scanner (Android) | Android 无障碍检查 |
| 对比度检查 | WCAG Contrast Checker | 颜色对比度验证 |
| 自动化 | flutter test + semantics | 语义树断言 |

## 实现步骤

### Phase 1：语义标注（2-3h）
1. 所有按钮/图标加 Semantics label
2. 图片加 semanticLabel（描述图片内容）
3. 自定义 Widget 加 Semantics 包装
4. 装饰性元素标记 excludeFromSemantics: true

### Phase 2：屏幕阅读器适配（2h）
5. 开启 TalkBack/VoiceOver 实际体验完整流程
6. 修复焦点顺序（确保逻辑顺序，不是布局顺序）
7. 自定义操作：滑动手势 → 语义动作映射
8. 动态内容变化时通知阅读器（SemanticsService.announce）

### Phase 3：视觉适配（2h）
9. 大字体测试：textScaleFactor=2.0 时布局不溢出
10. 高对比度：所有文字/背景对比度 ≥ 4.5:1（WCAG AA）
11. 色盲模式：状态不仅靠颜色（加图标/文字/形状区分）
12. 暗色模式：确保暗色下也满足对比度

### Phase 4：交互适配（1-2h）
13. 触摸目标：所有可点击区域 ≥ 48x48dp
14. 动画：尊重 MediaQuery.disableAnimations
15. 超时：给用户足够时间完成操作（不自动消失的提示）

## 关键坑点
| 坑 | 后果 | 解法 |
|----|------|------|
| 忘记标注 | 阅读器读出"按钮"而非功能 | 代码审查 + lint 规则 |
| 焦点陷阱 | 用户被困在某个区域 | 测试完整导航流程 |
| 动态内容 | 阅读器不知道内容变了 | announce + live region |
| 自定义绘制 | Canvas 绘制的内容无语义 | 加 Semantics 覆盖层 |
| 大字体溢出 | 文字被截断 | Flexible + overflow 处理 |
| 测试不足 | 以为做了但实际不可用 | 真实辅助技术测试 |
