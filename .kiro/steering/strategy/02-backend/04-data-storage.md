# 4. 后端数据存储

## 是什么
数据库选型、表结构设计、读写分离、缓存策略、数据备份。覆盖关系型（PostgreSQL）、文档型（MongoDB）、缓存（Redis）、对象存储（OSS）的选型决策和使用场景。

## 为什么需要
- 所有后端服务的基础——没有存储就没有状态
- 不同数据类型需要不同存储引擎（结构化/非结构化/时序/缓存）
- 性能瓶颈 90% 出在数据库
- 数据丢失 = 公司倒闭级事故

## 技术架构
```
┌─────────────────────────────────────────────────┐
│              数据存储分层                         │
├─────────────────────────────────────────────────┤
│                                                  │
│  [热数据] Redis                                  │
│   Session / Token / 计数器 / 排行榜             │
│   特点：内存级速度，TTL 自动过期               │
│                                                  │
│  [温数据] PostgreSQL                             │
│   用户 / 设备 / 订单 / 配置                    │
│   特点：ACID 事务，复杂查询，关系完整性        │
│                                                  │
│  [冷数据] 对象存储 (阿里云 OSS)                 │
│   固件文件 / 用户头像 / 日志归档               │
│   特点：无限容量，按量付费，CDN 加速           │
│                                                  │
│  [时序数据] TimescaleDB / InfluxDB              │
│   设备遥测 / 传感器数据 / 监控指标             │
│   特点：高写入吞吐，时间范围查询优化           │
│                                                  │
└─────────────────────────────────────────────────┘
```

## 技术栈选型
| 场景 | 技术 | 理由 |
|------|------|------|
| 核心业务 | PostgreSQL 15+ | ACID、JSON 支持、扩展生态 |
| 缓存 | Redis 7 | 亚毫秒延迟、丰富数据结构 |
| 文件存储 | 阿里云 OSS | 无限容量、CDN、低成本 |
| 时序数据 | TimescaleDB | PostgreSQL 扩展，无需新运维 |
| ORM | Prisma / TypeORM / GORM | 类型安全、迁移管理 |
| 备份 | pg_dump + WAL 归档 | 点恢复（PITR） |

## 实现步骤

### Phase 1：PostgreSQL 基础（2-3h）
1. 数据库设计原则：第三范式 → 适度反范式（性能）
2. 索引策略：B-tree（等值/范围）、GIN（JSONB/全文）、部分索引
3. 连接池：PgBouncer（transaction mode），防止连接耗尽
4. 迁移管理：版本化 SQL 文件，CI 自动执行

### Phase 2：Redis 缓存策略（2h）
5. 缓存模式：Cache-Aside（读时填充）、Write-Through（写时更新）
6. 过期策略：TTL 设置、LRU 淘汰、主动失效（数据变更时删缓存）
7. 常见用途：Session、限流计数器、排行榜（Sorted Set）、分布式锁

### Phase 3：对象存储（1-2h）
8. 上传流程：客户端直传 OSS（STS 临时凭证）→ 回调通知后端
9. CDN 加速：静态资源走 CDN，减少源站压力
10. 生命周期：自动归档（30天后转低频存储）

### Phase 4：备份与恢复（1-2h）
11. 自动备份：每日全量 pg_dump + 持续 WAL 归档
12. 恢复演练：每月一次恢复到测试环境验证
13. 多地备份：主库在华东，备份存华南

## 关键坑点
| 坑 | 后果 | 解法 |
|----|------|------|
| N+1 查询 | 列表页面几百次 DB 查询 | ORM eager loading / DataLoader |
| 缓存穿透 | 查不存在的 key 打穿 DB | 布隆过滤器 / 空值缓存 |
| 缓存雪崩 | 大量 key 同时过期 | TTL 加随机偏移 |
| 大事务 | 锁表导致其他请求超时 | 拆分事务、乐观锁 |
| 连接泄漏 | 连接池耗尽，服务不可用 | 连接池监控 + 超时回收 |
| 备份未验证 | 需要恢复时发现备份损坏 | 定期恢复演练 |
