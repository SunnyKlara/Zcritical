# 33. 文件存储服务

## 是什么
管理用户上传的文件（Logo 图片、音频素材、头像、固件包）的完整服务。包括上传、存储、CDN 分发、权限控制、生命周期管理。

## 为什么需要
- 用户自定义 Logo 需要上传图片
- 自定义音频需要上传音频文件
- OTA 固件包需要安全存储和分发
- 用户头像、社区图片等 UGC 内容
- 文件不能存数据库（性能差、成本高）

## 技术架构
```
┌─────────────────────────────────────────────────┐
│              文件存储服务                         │
├─────────────────────────────────────────────────┤
│                                                  │
│  [上传路径]                                      │
│   App → 后端获取 STS 临时凭证                   │
│       → 直传 OSS（不经过后端，省带宽）          │
│       → OSS 回调通知后端 → 记录元数据           │
│                                                  │
│  [存储层]                                        │
│   阿里云 OSS / AWS S3                           │
│   ├─ /avatars/     用户头像                     │
│   ├─ /logos/       设备 Logo                    │
│   ├─ /audio/       自定义音频                   │
│   ├─ /firmware/    固件包（私有读）             │
│   └─ /temp/        临时文件（7天自动删除）      │
│                                                  │
│  [分发层]                                        │
│   CDN 加速 → 就近节点 → 缓存                   │
│   图片处理：缩略图、裁剪、水印（OSS 图片处理） │
│                                                  │
│  [安全层]                                        │
│   签名 URL（限时访问）/ Referer 白名单          │
│   文件类型校验 / 大小限制 / 病毒扫描           │
│                                                  │
└─────────────────────────────────────────────────┘
```

## 技术栈选型
| 组件 | 技术 | 说明 |
|------|------|------|
| 对象存储 | 阿里云 OSS | 国内首选，CDN 集成好 |
| 临时凭证 | STS (Security Token Service) | 客户端直传，不暴露 AK |
| CDN | 阿里云 CDN | 全国节点，HTTPS |
| 图片处理 | OSS 图片处理 | URL 参数即可缩放/裁剪 |
| 元数据 | PostgreSQL | 文件记录关联用户/设备 |

## 实现步骤

### Phase 1：OSS 配置（1h）
1. 创建 Bucket，设置跨域（CORS）
2. 目录规划（按类型分目录）
3. 生命周期规则（temp/ 7天删除，归档策略）

### Phase 2：上传流程（2-3h）
4. 后端 API：`POST /api/upload/token` → 返回 STS 临时凭证
5. App 端直传 OSS（flutter_oss_aliyun 或 HTTP PUT）
6. 上传完成回调 → 后端记录文件元数据
7. 文件校验：类型白名单、大小限制（图片<5MB，音频<20MB）

### Phase 3：访问控制（1-2h）
8. 公开文件：头像、Logo → CDN 直接访问
9. 私有文件：固件包 → 签名 URL（有效期 1 小时）
10. 防盗链：Referer 白名单 + 签名验证

### Phase 4：图片处理（1h）
11. 缩略图：`?x-oss-process=image/resize,w_200`
12. 格式转换：PNG→WebP（省 30% 带宽）
13. Logo 适配：上传后自动生成设备端尺寸（240x240 RGB565）

## 关键坑点
| 坑 | 后果 | 解法 |
|----|------|------|
| AK 泄露 | 存储桶被清空/滥用 | STS 临时凭证，最小权限 |
| 大文件超时 | 上传失败 | 分片上传 + 断点续传 |
| 存储成本失控 | 月账单暴涨 | 生命周期规则 + 低频存储 |
| CDN 缓存脏数据 | 用户看到旧图片 | 文件名带 hash，更新即新 URL |
| 恶意文件 | 上传病毒/违规内容 | 类型校验 + 内容审核 API |

## 与 RideWind 的关系
- 当前状态：Logo 通过 BLE 直传设备（无云端存储）
- 优先级：P4（有后端后逐步迁移）
- 场景：Logo 云端备份、自定义音频库、OTA 固件托管
- 成本：OSS 标准存储 ¥0.12/GB/月，CDN ¥0.24/GB 流量
