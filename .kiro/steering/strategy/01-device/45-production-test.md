# 45. 产测系统

## 是什么

产线质量检测系统。每台设备出厂前自动跑一遍硬件自检，逐项验证所有外设是否正常工作。PASS 才能出货，FAIL 标记返修。包括设备端自检固件、产线工具软件、测试数据记录。

## 为什么需要

- 批量生产时焊接不良率 2-5%（虚焊、短路、漏焊）
- 不检测就出货 = 用户收到坏品 = 退货 + 差评 + 品牌损失
- 人工逐项检测太慢（每台 5-10 分钟），自动化 30 秒搞定
- 测试数据留档：追溯问题批次、统计良率、改进工艺
- 售后维修时可重跑自检定位故障

## 技术架构

```
┌─────────────────────────────────────────────────────┐
│                  产测系统架构                         │
├─────────────────────────────────────────────────────┤
│                                                      │
│  [设备端] selftest.c                                │
│   开机按住按钮 → 进入自检模式                       │
│   逐项测试 → LCD 显示 PASS/FAIL → 串口输出日志     │
│   ALL PASS → NVS 写标志 → 重启进入正常模式         │
│                                                      │
│  [产线工具]（可选，未来扩展）                       │
│   PC 端 Python/Electron 程序                        │
│   串口连接 → 读取测试结果 → 记录数据库             │
│   扫码枪关联设备 SN → 测试报告归档                 │
│                                                      │
│  [数据层]（可选，未来扩展）                         │
│   SQLite/云数据库 → 良率统计 → 异常报警            │
│                                                      │
└─────────────────────────────────────────────────────┘

测试项目：
  ┌────────────────────────────────────────┐
  │  1. LCD 显示    → 画彩色条纹验证       │
  │  2. LED Main    → 全亮红绿蓝验证       │
  │  3. LED Tail    → 全亮红绿蓝验证       │
  │  4. 编码器旋转  → 等待用户转动确认     │
  │  5. 编码器按键  → 等待用户按下确认     │
  │  6. 喇叭       → 播放测试音验证       │
  │  7. 风扇       → 开启验证转动         │
  │  8. 雾化器     → 开启验证工作         │
  │  9. BLE        → 广播验证（可跳过）   │
  │ 10. PSRAM      → 读写校验             │
  └────────────────────────────────────────┘
```

## 技术栈选型

| 组件 | 技术 | 说明 |
|------|------|------|
| 自检固件 | 集成在主固件中（条件进入） | 不需要单独烧录测试固件 |
| 进入方式 | 开机时按住编码器按钮 | 简单可靠，无需额外工具 |
| 结果显示 | LCD 逐项 PASS/FAIL + 最终汇总 | 产线工人一眼看结果 |
| 串口日志 | UART 115200 详细输出 | 接 PC 可自动化采集 |
| 产测锁 | NVS `selftest.passed=1` | 防止重复进入浪费时间 |
| 产线工具 | Python + pyserial（未来） | 自动化数据采集 |

## 实现步骤

### Phase 1：设备端自检（已完成）

1. **进入条件**
   - 开机时 GPIO 按钮为低电平 → 检查 NVS 产测锁
   - 未通过产测 → 进入自检模式
   - 已通过产测 → 正常启动

2. **测试项实现模式**
   - 每项独立函数，返回 bool（PASS/FAIL）
   - 逐项执行，LCD 实时显示当前项和结果
   - 串口同步输出详细日志

3. **各项测试逻辑**
   - LCD：填充红/绿/蓝全屏，目视确认无坏点
   - LED：逐颗点亮 R/G/B，目视确认
   - 编码器：等待旋转事件（超时 10s = FAIL）
   - 按键：等待按下事件
   - 喇叭：播放 1kHz 正弦波 1 秒
   - 风扇：GPIO 拉高，听/看是否转动
   - 雾化器：GPIO 拉高，看是否出雾
   - BLE：启动广播，检查是否成功（不需要连接）
   - PSRAM：写入 pattern → 读回比较

### Phase 2：产测锁与售后重测（已完成）

4. **产测锁机制**
   - ALL PASS 后写 NVS 标志
   - 后续开机即使按住按钮也不再进入自检
   - 节省产线时间

5. **售后重测入口**
   - BLE 命令清除 NVS 标志
   - 或全擦 NVS（会丢失所有设置）
   - 维修人员可重新跑自检定位故障

### Phase 3：产线工具软件（未来，2-3h）

6. **Python 自动化脚本**
   - 串口监听测试结果
   - 解析每项 PASS/FAIL
   - 关联设备 SN（扫码枪输入）
   - 保存测试报告（JSON/CSV）

7. **数据记录**
   - 每台设备：SN + 测试时间 + 各项结果 + 固件版本
   - 存储：本地 SQLite（小批量）或云端 API（大批量）
   - 统计：日良率、周趋势、故障项分布

### Phase 4：高级功能（未来，3-4h）

8. **自动化测试夹具**
   - 弹簧针（Pogo Pin）接触测试点
   - 自动供电 + 串口连接
   - 光传感器检测 LCD/LED 亮度
   - 麦克风检测喇叭输出
   - 电流表检测各外设功耗

9. **批次追溯**
   - 设备 SN 烧入 eFuse（不可改）
   - 关联：SN → 测试报告 → 生产批次 → 元器件批次
   - 发现问题时可精确召回特定批次

10. **良率看板**
    - 实时显示：今日测试数 / 通过数 / 良率
    - 异常报警：某项连续 3 台 FAIL → 通知工程师
    - 趋势分析：良率下降 → 检查焊接工艺

## 关键坑点

| 坑 | 后果 | 解法 |
|----|------|------|
| 测试顺序不对 | 前项 FAIL 影响后项 | 独立测试，每项自行初始化/清理 |
| 超时设置不当 | 工人等太久 / 误判 FAIL | 每项 5-10s 超时，可调 |
| 目视项不可靠 | 工人疲劳漏检 | 未来加传感器自动判定 |
| 产测锁误锁 | 未测完就锁了 | 只在 ALL PASS 后才写锁 |
| 串口日志丢失 | 无法追溯 | PC 端实时保存 + 云端备份 |
| 固件版本不匹配 | 测试项与硬件不对应 | 自检代码读取硬件版本号适配 |
| 环境干扰 | BLE 测试在产线环境不稳定 | BLE 标记为可跳过 |
