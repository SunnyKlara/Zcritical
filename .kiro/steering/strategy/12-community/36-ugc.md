# 36. 用户 UGC 分享

## 是什么
用户创建并分享自定义内容（灯效方案、音效配置、骑行数据），其他用户可以浏览/下载/使用。是"用户为用户创造价值"的内容生态，让产品从"工具"变成"平台"。

## 为什么需要
- 丰富产品内容生态：官方做 10 个灯效，用户社区可以做 1000 个
- 增加用户粘性：创作者有成就感，消费者有新鲜感
- 降低新用户上手门槛：一键导入别人的配置，不用自己从零调
- 网络效应：用户越多 → 内容越多 → 吸引更多用户
- 差异化竞争：竞品没有 UGC 生态

## 技术架构
```
┌─────────────────────────────────────────────────────┐
│              UGC 分享系统架构                         │
├─────────────────────────────────────────────────────┤
│                                                      │
│  [创作端]                                            │
│   用户在 App 中创建内容 → 保存为标准格式           │
│   ├─ 灯效方案：颜色+模式+速度+亮度 参数组合       │
│   ├─ 音效配置：引擎音+音量+混音比例               │
│   └─ 设备配置：全套参数快照                        │
│       │                                              │
│       ▼                                              │
│  [发布流程]                                          │
│   命名 + 描述 + 预览图/视频 → 审核 → 上架         │
│       │                                              │
│       ▼                                              │
│  [内容商店]                                          │
│   浏览 / 搜索 / 分类 / 排行 / 推荐                │
│   热门 / 最新 / 编辑推荐 / 关注的人               │
│       │                                              │
│       ▼                                              │
│  [消费端]                                            │
│   预览效果 → 下载 → 一键应用到设备                 │
│       │                                              │
│       ▼                                              │
│  [反馈循环]                                          │
│   评分 / 评论 / 下载量 → 创作者激励               │
│                                                      │
└─────────────────────────────────────────────────────┘
```

## 技术栈选型
| 组件 | 技术 | 说明 |
|------|------|------|
| 内容存储 | 阿里云 OSS + PostgreSQL | 文件存 OSS，元数据存 DB |
| 内容格式 | JSON Schema 标准化 | 版本化，向后兼容 |
| 预览生成 | 服务端渲染 / App 截图 | 自动生成预览图 |
| 推荐算法 | 协同过滤 / 热度排行 | 初期用热度，后期加个性化 |
| 审核 | 自动规则 + 人工 | 防止恶意/违规内容 |
| CDN | 阿里云 CDN | 内容下载加速 |
| 搜索 | PostgreSQL FTS | 按名称/标签/作者搜索 |

## 实现步骤

### Phase 1：内容格式标准化（2h）
1. 灯效方案格式定义：
   ```json
   {
     "schema_version": "1.0",
     "type": "led_preset",
     "name": "赛博朋克",
     "description": "紫蓝渐变呼吸灯效",
     "author_id": "uuid",
     "preview_url": "https://...",
     "config": {
       "mode": "gradient_breath",
       "colors": ["#8B5CF6", "#3B82F6", "#06B6D4"],
       "speed": 75,
       "brightness": 80,
       "direction": "forward"
     },
     "compatibility": {
       "min_firmware": "1.2.0",
       "led_count_min": 30
     }
   }
   ```
2. 音效配置格式：引擎类型 + 音量曲线 + 混音参数
3. 设备配置快照：所有可配置参数的完整导出

### Phase 2：发布与审核（2-3h）
4. 发布流程：
   - 用户在 App 中调好效果 → "分享到社区"
   - 填写名称/描述/标签 → 自动生成预览
   - 提交审核 → 通过后上架
5. 审核规则：
   - 自动：名称/描述敏感词检测
   - 自动：配置参数合法性校验（值在有效范围内）
   - 人工：预览图/视频审核（首次发布）
6. 版本管理：创作者可更新已发布内容，保留历史版本

### Phase 3：内容商店（3-4h）
7. 浏览页面：
   - 分类 Tab：灯效 / 音效 / 全套配置
   - 排序：热门 / 最新 / 评分最高
   - 筛选：标签 / 兼容性 / 免费/付费
8. 详情页：
   - 预览（图片/视频/动画模拟）
   - 作者信息 + 下载量 + 评分
   - 评论区
   - "应用到设备"按钮
9. 搜索：关键词 + 标签 + 作者名

### Phase 4：一键应用（2h）
10. 下载流程：点击"应用" → 下载配置 → BLE 写入设备
11. 兼容性检查：固件版本/LED 数量/硬件型号是否匹配
12. 冲突处理：应用新配置前提示"将覆盖当前设置，是否备份？"
13. 回滚：应用后不满意 → 一键恢复之前的配置

### Phase 5：激励与推荐（1-2h）
14. 创作者激励：
    - 下载量展示（成就感）
    - 积分奖励（下载 1 次 = 创作者 +1 积分）
    - 编辑推荐（官方精选）
    - 未来：付费内容分成
15. 推荐算法：
    - 热度 = 下载量×1 + 点赞×2 + 评论×3（时间衰减）
    - 个性化：基于用户历史下载的标签偏好
16. 排行榜：周榜/月榜/总榜，激励持续创作

## 关键坑点
| 坑 | 后果 | 解法 |
|----|------|------|
| 内容质量参差 | 垃圾内容淹没优质内容 | 审核 + 评分 + 编辑推荐 |
| 格式不兼容 | 新版本配置旧设备用不了 | schema_version + 兼容性字段 |
| 抄袭/盗用 | 创作者积极性受挫 | 相似度检测 + 举报机制 |
| 冷启动 | 没有内容没人来 | 官方先发 20+ 优质预设 |
| 恶意配置 | 极端参数损坏设备 | 参数范围校验 + 安全限制 |
| 预览不准 | 实际效果和预览不一样 | 真机录制预览 / 精确模拟器 |

## 与 RideWind 的关系
- 当前状态：无 UGC 系统，灯效/音效只有出厂预设
- 优先级：P5（产品功能稳定 + 有用户基数后）
- 可能的 UGC 类型：
  - 灯效预设方案（颜色+模式组合）
  - 音效配置（引擎音+参数）
  - 全套设备配置快照
- 起步：App 内"预设分享"功能（不需要完整社区）
