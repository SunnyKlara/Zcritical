# 35. 社区/论坛系统

## 是什么
用户交流平台，分享使用心得、配置方案、问题求助、官方公告。让用户之间形成互助网络，降低官方客服压力，同时收集真实需求和创意，建立品牌社区文化。

## 为什么需要
- 降低客服压力：用户互助解决 60%+ 的常见问题
- 增加粘性：社区活跃用户留存率比普通用户高 3 倍
- 收集需求：用户自发讨论的功能需求最真实
- 建立品牌社区：从"买产品"到"加入社区"，情感连接更深
- 内容沉淀：问题解答沉淀为知识库，长期价值

## 技术栈选型
| 组件 | 技术 | 说明 |
|------|------|------|
| 自建前端 | React + Ant Design | App 内 WebView 嵌入 |
| 自建后端 | Node.js + PostgreSQL | 完全可控 |
| 开源方案 | Discourse | 功能最全，Ruby 生态 |
| 轻量方案 | Flarum | PHP，现代 UI，轻量 |
| 搜索 | PostgreSQL FTS / MeiliSearch | 初期 FTS 够用 |
| 图片存储 | 阿里云 OSS + CDN | 用户上传图片 |
| 审核 | 阿里云内容安全 API | 文本+图片审核 |

## 实现步骤

### Phase 1：核心数据模型（2-3h）
1. 帖子表设计：
   ```sql
   CREATE TABLE posts (
     id          UUID PRIMARY KEY,
     author_id   UUID REFERENCES users(id),
     board_id    UUID REFERENCES boards(id),
     title       VARCHAR(200),
     content     TEXT,
     images      JSONB,
     tags        VARCHAR(50)[],
     status      VARCHAR(20),  -- draft/published/hidden/deleted
     is_pinned   BOOLEAN DEFAULT false,
     is_featured BOOLEAN DEFAULT false,
     view_count  INT DEFAULT 0,
     like_count  INT DEFAULT 0,
     reply_count INT DEFAULT 0,
     created_at  TIMESTAMPTZ DEFAULT NOW(),
     updated_at  TIMESTAMPTZ DEFAULT NOW()
   );

   CREATE TABLE replies (
     id          UUID PRIMARY KEY,
     post_id     UUID REFERENCES posts(id),
     author_id   UUID REFERENCES users(id),
     content     TEXT,
     images      JSONB,
     like_count  INT DEFAULT 0,
     parent_id   UUID,  -- 楼中楼回复
     created_at  TIMESTAMPTZ DEFAULT NOW()
   );
   ```
2. 版块设计：使用心得 / 配置分享 / 问题求助 / 官方公告 / 功能建议
3. 标签系统：灯效/音效/速度/BLE/OTA/硬件

### Phase 2：帖子与互动（3-4h）
4. 发帖流程：选版块 → 写标题/内容 → 上传图片 → 选标签 → 发布
5. 回复系统：楼层回复 + 楼中楼（@某人）
6. 互动功能：点赞/收藏/分享/举报
7. 通知系统：有人回复你/点赞你/@你 → 推送通知

### Phase 3：内容管理（2h）
8. 精华帖：管理员标记优质内容，首页推荐
9. 置顶：版块内置顶重要帖子
10. 热门排行：按 (点赞数×2 + 回复数×3 + 浏览数×0.1) / 时间衰减
11. 搜索：标题+内容全文搜索，支持标签筛选

### Phase 4：审核与安全（2h）
12. 发布前审核：
    - 敏感词过滤（本地词库 + 云端 API）
    - 图片审核（阿里云内容安全）
    - 新用户前 3 帖需人工审核
13. 举报处理：用户举报 → 审核队列 → 处理（删除/警告/封禁）
14. 反垃圾：频率限制 + 重复内容检测 + IP 黑名单

### Phase 5：积分与激励（1-2h）
15. 积分规则：
    - 发帖 +5 / 回复 +2 / 被点赞 +1 / 签到 +1
    - 精华帖 +20 / 被采纳为最佳回答 +10
16. 等级体系：新手→入门→进阶→达人→专家（按积分升级）
17. 特权：高等级用户可以编辑标签、推荐帖子

## 关键坑点
| 坑 | 后果 | 解法 |
|----|------|------|
| 冷启动 | 没人发帖没人看 | 官方先发优质内容 + 种子用户邀请 |
| 内容质量低 | 水帖泛滥 | 积分激励优质内容 + 精华帖机制 |
| 审核成本 | 人工审核忙不过来 | AI 预审 + 社区举报 + 信任等级 |
| 负面内容 | 用户吐槽/攻击 | 引导建设性反馈 + 快速响应 |
| SEO 不友好 | App 内容搜索引擎抓不到 | 同步到 Web 版，SSR 渲染 |
| 数据迁移 | 从微信群迁移历史讨论 | 整理精华内容手动搬运 |

## 与 RideWind 的关系
- 当前状态：无社区，用户交流在微信群（信息不沉淀）
- 优先级：P5（用户量到百级后）
- 起步方案：微信群 → 飞书知识库（FAQ）→ App 内社区
- 依赖：#2 用户系统（发帖需要账号）
- 价值：即使 50 个用户，社区也能产生互助价值
