# 34. 用户行为分析

## 是什么
在基础埋点之上，深度分析用户在 App 中的行为路径、留存规律、转化漏斗、用户分群。不只是"看数据"，而是"从数据中发现洞察"——为什么用户流失？哪些用户最有价值？如何精细化运营？

## 为什么需要
- 埋点只是采集数据，行为分析才是从数据中提取价值
- 找到产品瓶颈：用户在哪一步放弃？为什么？
- 优化转化率：注册→绑定→活跃→付费，每一步都可以优化
- 精细化运营：不同用户群用不同策略（新用户引导 vs 老用户召回）
- 预测流失：识别即将流失的用户，提前干预

## 实现步骤

### Phase 1：漏斗分析（2-3h）
1. 定义核心漏斗：
   - 新用户漏斗：下载→启动→注册→绑定设备→首次使用功能
   - 功能漏斗：进入页面→操作→完成→再次使用
2. SQL 实现漏斗查询：
   ```sql
   -- 7天内完成各步骤的用户数
   WITH funnel AS (
     SELECT user_id,
       MAX(CASE WHEN event='app_open' THEN 1 END) as step1,
       MAX(CASE WHEN event='register' THEN 1 END) as step2,
       MAX(CASE WHEN event='device_bind' THEN 1 END) as step3,
       MAX(CASE WHEN event='feature_use' THEN 1 END) as step4
     FROM events WHERE ts > now() - interval '7 days'
     GROUP BY user_id
   )
   SELECT count(*) as total,
     sum(step1) as opened, sum(step2) as registered,
     sum(step3) as bound, sum(step4) as active
   FROM funnel;
   ```
3. 漏斗可视化：柱状图展示每步转化率和流失率

### Phase 2：留存分析（2h）
4. 留存定义：用户在第 N 天是否回访
5. 留存矩阵计算：
   ```sql
   -- 按注册日期分组的留存率
   SELECT register_date,
     count(DISTINCT user_id) as cohort_size,
     count(DISTINCT CASE WHEN day_diff=1 THEN user_id END) as d1,
     count(DISTINCT CASE WHEN day_diff=7 THEN user_id END) as d7,
     count(DISTINCT CASE WHEN day_diff=30 THEN user_id END) as d30
   FROM user_activity_daily
   GROUP BY register_date;
   ```
6. 留存曲线图：对比不同版本/渠道的留存差异
7. 留存驱动因素：D1 留存高的用户做了什么？（Aha Moment）

### Phase 3：用户分群（2-3h）
8. RFM 模型：
   - Recency（最近一次使用距今天数）
   - Frequency（过去 30 天使用次数）
   - Monetary（付费金额，或使用深度）
9. 行为标签：
   - 活跃度：日活/周活/月活/沉默/流失
   - 功能偏好：音效玩家/灯效玩家/速度玩家
   - 设备类型：单设备/多设备用户
10. 分群策略：
    - 高价值用户 → VIP 服务 + 新功能内测
    - 即将流失用户 → 召回推送 + 优惠券
    - 新用户 → 引导教程 + 功能推荐

### Phase 4：路径分析与归因（2h）
11. 用户路径：最常见的行为序列是什么？
    - 桑基图展示：首页→速度页→颜色页→退出
12. 异常路径：流失用户的最后行为是什么？
13. 归因分析：哪个功能/渠道对留存贡献最大？
    - 相关性分析：使用了功能 X 的用户留存率高 20%

## 关键坑点
| 坑 | 后果 | 解法 |
|----|------|------|
| 数据量不够 | 统计不显著，结论不可靠 | 等用户量到千级再做深度分析 |
| 相关≠因果 | 误判功能价值 | A/B 测试验证因果关系 |
| 分群太细 | 每群人太少无法运营 | 3-5 个核心分群足够 |
| 留存定义不统一 | 不同口径结果不同 | 文档化定义（日活=当天有任意事件） |
| 隐私问题 | 用户画像过于精细 | 匿名化 + 聚合分析 + 合规 |
| 分析瘫痪 | 看了很多数据不行动 | 每次分析必须输出 action item |
