# 26. 电商系统

## 是什么
在线销售产品的完整系统：商品展示、购物车、下单、支付、物流追踪、售后退款。让用户直接在 App/小程序/官网购买 RideWind 产品，实现 DTC（Direct-to-Consumer）模式，减少中间商，掌握用户数据。

## 为什么需要
- 当前通过淘宝/拼多多销售，平台抽佣 3-5%，且不掌握用户数据
- DTC 模式：直接面向消费者，利润更高，用户关系更近
- 数据闭环：购买→使用→复购，全链路数据可追踪
- 品牌建设：自有渠道 = 品牌形象可控
- 配件/耗材复购：音效包、灯效包、替换配件的持续收入

## 技术栈选型
| 组件 | 技术 | 说明 |
|------|------|------|
| 自建后端 | Node.js/Go + PostgreSQL | 完全可控 |
| 轻量方案 | Shopify API + 自定义前端 | 快速上线，运维少 |
| 微信生态 | 微信小商店 / 有赞 | 国内社交电商 |

| 搜索 | Elasticsearch / PostgreSQL FTS | 商品搜索 |

## 实现步骤

### Phase 1：商品管理（2-3h）
1. 数据模型设计：
   ```sql
   -- SPU (Standard Product Unit) 标准产品
   CREATE TABLE products (
     id          UUID PRIMARY KEY,
     name        VARCHAR(200),
     description TEXT,
     images      JSONB,        -- ["url1", "url2"]
     category    VARCHAR(50),
     status      VARCHAR(20),  -- draft/active/archived
     created_at  TIMESTAMPTZ DEFAULT NOW()
   );
   
   -- SKU (Stock Keeping Unit) 库存单位
   CREATE TABLE skus (
     id          UUID PRIMARY KEY,
     product_id  UUID REFERENCES products(id),
     name        VARCHAR(200),  -- "RideWind Pro 黑色"
     price       DECIMAL(10,2),
     stock       INT DEFAULT 0,
     attributes  JSONB,         -- {"color": "black", "version": "pro"}
     status      VARCHAR(20)
   );
   ```
2. 商品管理后台：CRUD + 图片上传 + 上下架
3. 库存管理：入库/出库/预占/释放

### Phase 2：购物车与订单（3-4h）
4. 购物车：
   - Redis 存储（登录用户）+ 本地存储（游客）
   - 加入/修改数量/删除/清空
   - 库存校验：加入时检查，下单时再次检查
5. 订单状态机：
   ```
   待支付 → 已支付 → 待发货 → 已发货 → 已签收 → 已完成
     │                                              │
     └── 已取消（超时未支付）          已退款 ←── 退款中
   ```
6. 下单流程：
   - 购物车 → 确认订单（地址/优惠券）→ 创建订单 → 锁定库存
   - 支付成功回调 → 更新订单状态 → 通知发货
   - 超时未支付（30min）→ 自动取消 → 释放库存

### Phase 3：物流集成（1-2h）
7. 发货：后台填写快递单号 → 更新订单状态
8. 物流追踪：快递 100 API 订阅 → 状态变更推送
9. 用户端展示：物流时间线（揽收→运输→派送→签收）
10. 签收确认：物流显示签收 → 7 天后自动确认收货

### Phase 4：售后退款（2h）
11. 退款申请：用户提交原因 + 图片
12. 审核流程：自动审核（金额<50）/ 人工审核（金额>=50）
13. 退款执行：调用支付平台退款 API → 原路退回
14. 退货物流：生成退货地址 → 用户寄回 → 确认收货 → 退款

### Phase 5：营销工具（2h）
15. 优惠券：满减/折扣/新人券，有效期/使用条件
16. 限时活动：秒杀/团购/预售
17. 会员价：不同等级不同价格

## 关键坑点
| 坑 | 后果 | 解法 |
|----|------|------|
| 超卖 | 库存为负，无法发货 | Redis 原子扣减 + 数据库乐观锁 |
| 重复支付 | 一个订单付两次钱 | 订单号幂等 + 支付状态校验 |
| 订单超时 | 库存被长期占用 | 延迟队列 30min 自动取消 |
| 并发下单 | 同一商品多人抢购 | 分布式锁 / Redis 预扣库存 |
| 退款纠纷 | 用户投诉/差评 | 清晰退款政策 + 快速响应 |
| 数据一致性 | 支付成功但订单未更新 | 支付回调重试 + 对账 |

## 与 RideWind 的关系
- 当前状态：通过淘宝/拼多多销售，无自有电商
- 优先级：P5（规模化后自建，初期用第三方平台）
- 演进路径：淘宝 → 微信小商店 → 自建商城
- 自建时机：月销 >500 台，平台佣金 > 自建成本时
- 配件商城：音效包/灯效包/替换件可以先在 App 内售卖
