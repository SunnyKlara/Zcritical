# Git Commit 规范

## 格式

```
类型(模块): 标题（一句话概括）

A. 具体改动描述
B. 具体改动描述
C. 涉及文件/原因/影响
```

## 类型（纯中文）

| 类型 | 用途 |
|------|------|
| 新增 | 新功能、新文件、新模块 |
| 修复 | Bug 修复 |
| 优化 | 性能优化、体验改善（不改功能） |
| 调参 | 参数微调（亮度、速度、间距等） |
| 重构 | 代码结构调整（不改功能） |
| 文档 | 文档新增/更新 |

## 模块

| 模块 | 范围 |
|------|------|
| 固件 | ridewind-esp 下所有代码 |
| APP | RideWind Flutter 应用 |
| 全局 | 跨模块改动、工程配置、CI |

## 示例

```
新增(固件): 波浪LED效果 — 连续正弦波从左到右扫过6颗Main灯珠

A. led_effects.c 新增 throttle_fx_wave 函数
B. 参数：底亮15%(38)，峰值100%(255)，周期2.5s，正弦S曲线
C. ui_speed.c 进入Speed界面时自动启动，退出时停止
D. 尾灯跟随Main但相位偏移60单位（约600ms延迟）
```

```
修复(APP): RGB详情页LMRB末尾B胶囊右侧溢出7.2px

A. colorize_rgb_detail_view.dart 第89行 horizontalPadding 计算修正
B. 原因：screenWidth * 0.035 在窄屏(360px)下不够容纳4个胶囊
C. 修复：改用固定 padding 12px
```

```
调参(固件): 波浪效果波谷亮度30%→15%

A. led_effects.c BASE_BRIGHT 76→38
B. 用户反馈亮暗对比不够明显，降低底亮增强起伏感
```

## 规则

1. 标题不超过 50 个中文字符（git log --oneline 可读）
2. 正文用 A/B/C 分点，每点说清"改了什么"+"为什么改"
3. 涉及多文件时列出关键文件名
4. 参数变化写明 旧值→新值
5. 不混用中英文类型前缀（统一用中文）
