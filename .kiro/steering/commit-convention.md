# Git Commit 规范 + AI 自主执行规则

## Commit 格式

```
类型(模块): 标题（一句话概括）

A. 具体改动描述
B. 具体改动描述
C. 涉及文件/原因/影响
```

## 类型（纯中文）

| 类型 | 用途 |
|------|------|
| 新增 | 新功能、新文件、新模块 |
| 修复 | Bug 修复 |
| 优化 | 性能优化、体验改善（不改功能） |
| 调参 | 参数微调（亮度、速度、间距等） |
| 重构 | 代码结构调整（不改功能） |
| 文档 | 文档新增/更新 |

## 模块

| 模块 | 范围 |
|------|------|
| 固件 | ridewind-esp 下所有代码 |
| APP | RideWind Flutter 应用 |
| 全局 | 跨模块改动、工程配置、CI |

## 规则

1. 标题不超过 50 个中文字符
2. 正文用 A/B/C 分点，说清"改了什么"+"为什么改"
3. 涉及多文件时列出关键文件名
4. 参数变化写明 旧值→新值
5. 不混用中英文类型前缀（统一用中文）

## AI 自主执行规则

当已有完善的 git 分支/tag 管理时，AI 直接执行实验性改动，不等用户确认。

流程：
1. 说明方案（1-2句）
2. 保存当前版本（tag）
3. 直接执行改动 + 编译
4. 呈现结果供用户评判

前提：
- 已在实验分支上（不影响主干）
- 有 tag 可随时回退
- 不涉及破坏性操作（删分支、force push）

用户只需看结果说"好"或"不好"，不需要中间确认步骤。
